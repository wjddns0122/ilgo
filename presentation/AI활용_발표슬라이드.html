Prompt Harness

AI가 마음대로 말하지 못하게 — 답을 강제하는 설계

취약계층에게 틀린 답은 불편이 아니라 피해입니다. 프롬프트를 ‘계약’으로 만들었습니다.

① JSON 스키마 강제

응답이 "{" 로 시작하도록 강제(prefill·구조화 출력 모드)해 JSON 외의 답을 원천 차단. temperature 0.2로 형식 이탈 최소화. 파싱 실패 시 자동 재시도 → 그래도 실패하면 422로 명확히 격리.

② 사실 통제 규칙 (시스템 프롬프트)

“금액·날짜는 원문 그대로 인용 / 불확실하면 null / 없는 사실 생성 금지.” 환각을 스키마 수준에서 봉쇄합니다.

③ 안전 편향 — 확신 없으면 노랑

위험 판정이 애매하면 무조건 yellow로 상향 — 사기를 ‘안전’으로 놓치는 false negative를 구조적으로 방지. 스미싱·임금차감·부당공제·과태료·신고기한을 위험 유형으로 명시.

④ 실패까지 설계 — 모델 이중화

메인 gemini-3-flash-preview 장애 시 gpt-5.4로 자동 전환, 그마저 실패(502)하면 폴백 캐시 + ‘예시’ 배지로 응답 — 발표 데모가 절대 멈추지 않습니다.

// 프론트·백엔드가 공유하는 데이터 계약 — ReadResult { "doc_type": "notice", "summary_one_line": "2월 전기요금 62,210원을 3월 5일까지 내야 합니다.", "explained_text": "…쉬운 말 풀이…", "cards": { "what", "when", "where", "amount", "deadline" }, "risks": [{ "level": "green|yellow|red", "type", "message" }], "actions": [{ "text", "priority" }], "reply_drafts": [{ "korean", "note_in_lang" }], "consequence": "안 내면 연체료가 붙어요." }

이 스키마 하나가 프롬프트 출력 = API 응답 = Flutter freezed 모델로 그대로 이어집니다. AI 출력이 곧 타입 안전한 코드가 되는 구조.
