## 1. 정체성

| 항목 | 값 |
|---|---|
| 이름 | **또바기** (또박또박 → 또바기, "또박또박 읽어주는 친구") |
| 컨셉 | 편지·문서에서 태어난 작고 둥근 친구. 손에 작은 램프(뜻을 밝혀줌) |
| 형태 | 비인간·둥근 형태. 사람 형상 회피(인종·성별·장애 재현 함정 회피). 작은 크기에서도 식별되는 큰 눈 |
| 성격 | 다정, 느림, 또박또박. 절대 다그치지 않음 |
| **말투(상황별 혼합)** | **평소 = 다정한 반말**("어려운 글 오면 보여줘", "잘했어"). **위험·피해 안내 = 존댓말**("위험할 수 있어요", "지금 바로 112에 전화해요") |
| 색 정체성 | 기본 중립색(파랑/민트 계열). **볼(cheek) 색만 신호등과 연동**(초/노/빨) |

### 말투 규칙 (구현 기준)
- 반말 컨텍스트: onboarding, home, loading, result 낭독, safe, cheer, share.
- 존댓말 컨텍스트: `risks[].level ∈ {red, yellow}` 경고, 스미싱 피해대응(§4.7).
- 판정: `buildTobagiLine`이 컨텍스트 태그로 반말/존댓말 템플릿을 분기(§5).
- **영어(마리아, `lang=en`) 출력:** 반말/존댓말 구분이 없다 → **따뜻하고 명확한 단일 톤**으로 통일. 상황별 혼합(반말/존댓말)은 **한국어 출력에만** 적용한다.

## 2. 상태 머신 (애니 상태 — 경량)

```text
        온보딩/홈
          │
      ┌── idle ──────────────┐
      │     │                │
   분석시작  │             할일완료
      ▼     │                ▼
  thinking  │              cheer ──▶ idle
      │     │
    결과도착 │
      ▼     ▼
   reading ─┴─▶ (위험도 분기)
                 ├─ red  ▶ warn  (볼 빨강, 눈썹 걱정)
                 ├─ yellow ▶ caution (볼 노랑)
                 └─ green ▶ safe  (볼 초록, 미소)
```

| 상태 | 트리거 | 시각 | 음성 |
|---|---|---|---|
| idle | 홈/대기 | 눈 깜빡, 살짝 흔들 | 없음(또는 인사 1회) |
| thinking | 분석 로딩 | 램프 깜빡 | "지금 읽고 있어…" |
| reading | 결과 등장 + **탭** | 램프 밝아짐, 입 움직임 | 한 줄 요약 낭독 |
| warn | `level==red` | 볼 빨강, 눈썹 ↓ | 존댓말 경고 + risk message |
| caution | `level==yellow` | 볼 노랑 | 존댓말 "한번 확인해요" |
| safe | 최고 위험 green/∅ | 볼 초록, 미소 | 반말 "이건 안전해 보여" |
| cheer | 할 일 체크 완료 | 점프/반짝 | 반말 "잘했어! 하나 끝냈다" |

> 구현 최소치: idle / thinking / warn / safe / cheer 5종. caution은 warn 색만 노랑으로 재사용.

## 3. 음성 (TTS)

- 엔진: 기존 `TtsService`(flutter_tts) 재사용. 새 의존성 없음.
- 언어: `output_mode/lang`에 맞춰 ko / en. 민수=한국어 또바기, 마리아=영어 또바기.
- 파라미터: 느린 속도(rate ↓), 명확한 피치. 설정 속도값 공유.
- **기본 동작 = 탭하면 재생** (결과화면). 또바기를 탭해야 낭독 시작. **자동재생 아님.**
  - 설정 토글 "결과를 열면 자동으로 읽기"로 자동재생 선택 가능(기본 off).
  - 근거: 심사장·조용한 환경 배려 + 사용자 주도권. 발표자가 원할 때만 탭.
- 낭독은 **짧게**(한 줄 요약·경고문). 원문 전체 낭독은 기존 `ListenTtsView` 담당(중복 금지).
- 음소거 토글: 설정 + 또바기 롱프레스. 음소거 시 대사는 **자막(텍스트)**으로만.

## 4. 화면별 등장 + 대사 스크립트

> 모든 대사는 사전 작성 템플릿. `{ }`는 ReadResult에서 채움. 라이브 생성 아님. **[반]=반말 / [존]=존댓말.**

### 4.1 온보딩
```
[반] (첫 등장) "안녕! 나는 또바기야. 어려운 글이 오면 나한테 보여줘. 내가 쉽게 읽어줄게."
[반] (글씨 크기 단계, E3) "글씨 잘 보여? 크게도 할 수 있어. 슬라이더를 움직여봐."
[반] (모드 선택 후) "좋아. 이제 사진 한 장이면 돼."
```
> E3: 이 단계에 **실시간 글씨 크기 미리보기 문장**이 함께 뜬다(§통합기능정의 E3). 기존 text-size 뷰 + shared_preferences 재사용.

### 4.2 홈 (idle)
```
[반] (진입 1회) "오늘도 어려운 글 있으면 찍어줘."
[반] (예시 카드 옆) "처음이면 '예시로 체험하기'부터 해봐."
```

### 4.3 분석 로딩 (thinking)
```
[반] "지금 읽고 있어… 조금만 기다려줘."
```

### 4.4 결과 등장 (reading) — 주무대
```
[반] (또바기 탭 시) "이 글은 이런 뜻이야."  → 이어서 {summary_one_line} 낭독
```
> 결과 화면이 또바기의 히어로 무대. **레이아웃: 기존 TTS 낭독(`ListenTtsView`)은 그대로 두고, 그 아래에 또바기 "쉬운 보기" 섹션**(캐릭터 + 큰 글씨 한 줄 요약 + 탭하면 듣기 + 볼색)을 얹는다. **① 들려주기(기존 TTS) + ② 쉽게 보여주기(또바기)** 이중 채널. 위험도에 따라 볼 색이 바뀌어 **읽지 않아도 위험을 색으로 인지**. 또바기는 기존 기능을 대체하지 않는다.

### 4.5 위험 분기 (존댓말)
```
[존] red:    "잠깐! 이거 위험할 수 있어요. {risks[0].message} 링크가 있으면 누르지 마세요."
[존] yellow: "이건 한번 확인해보는 게 좋아요. {risks[0].message}"
[반] green:  "이건 안전해 보여. 걱정 마."
```

### 4.6 할 일 체크 (cheer, 반말)
```
[반] (체크 시) "잘했어! 하나 끝냈다."
[반] (전부 완료) "다 했어! 정말 잘했어."
```

### 4.7 스미싱 피해대응 (존댓말)
```
[존] (피해 의심 진입)
"혹시 벌써 링크를 눌렀거나 돈을 보내셨어요? 괜찮아요, 지금부터 같이 해요."
"돈을 보내셨으면 지금 바로 112에 전화하고, 보낸 은행에 '지급정지'를 요청해요."
"돌려받을 수 있는 공식 절차가 있어요. 어디에 연락할지 알려드릴게요."
```
> 톤 규칙: "돌려준다/보상한다" 단정 금지 → "돌려받을 수 있는 공식 절차가 있어요"(안내).

### 4.8 답장 (native / 마리아, 반말)
```
[반] "정중한 한국어로 답장을 만들어봤어. 그대로 보내도 돼."
[반] (재생성) "더 정중하게 다시 써볼게."
```

### 4.9 공유 (반말)
```
[반] "이 내용을 가족이나 동료에게 '같이 확인해줘'라고 보낼 수 있어."
```

## 5. 대사 생성 규칙 (파생 함수)

```text
buildTobagiLine(context, readResult) -> {state, text, register(반말|존댓말), speak:bool}

- onboarding      -> 반말, 고정 인사(+E3 글씨크기 안내)
- loading         -> 반말, 고정 "지금 읽고 있어…"
- result(탭)      -> state=reading, register=반말, text=summary_one_line, speak=true
- topRisk = max(readResult.risks by level)
    level red     -> state=warn,   register=존댓말, text=경고 템플릿 + topRisk.message
    level yellow  -> state=caution, register=존댓말
    level green/∅ -> state=safe,   register=반말
- actionDone      -> state=cheer, register=반말, 응원 템플릿
- smishingVictim  -> register=존댓말, 고정 피해대응 스크립트(§4.7)
                     트리거: 사기/스미싱 red 위험상세의 '②벌써 당했다면' CTA를 사용자가 탭할 때(자동 아님)
```

- 위험 타입 매칭: 완전일치 대신 포함검색 fallback(스미싱 PRD §9.2 표준화 재사용).
- `risks`가 비면 → safe(반말) 처리, 위험 대사 생략.

## 6. 접근성 규칙

- **강요 금지:** 또바기는 항상 스킵/음소거 가능. 화면 조작을 막지 않는다.
- **음성 탭재생:** 자동으로 떠들지 않음. 사용자가 탭할 때만.
- 터치 타깃 48dp+. 또바기 탭 = 낭독, 롱프레스 = 음소거 메뉴.
- 고대비: 볼 신호등 색은 텍스트 위험 배지와 **중복 정보**(색만으로 정보 전달 금지 — 대사·아이콘 병행).
- 자막: 또바기 대사는 화면에도 텍스트로 표시(청각 의존 금지).
- 모션 민감: "애니 줄이기" 설정 시 정적 이미지로 폴백.
- 글씨 크기: 또바기 대사 자막도 앱 글씨 크기 설정(E3)을 따른다.
