## 1. 공통 API 계약 (프론트와 동일 — 이 계약만 지키면 병렬 가능)

### 1-1. 기본

- Base URL: `ilgo-backend-production.up.railway.app` (프론트가 `--dart-define=BASE_URL` 주입)
- 헤더: `Content-Type: application/json`, `Accept-Language: <ko|en|km|vi|ne|zh|th|id>`, 로그인 후 `Authorization: Bearer`, 로그인 전 `X-Device-Id`.
- 시간 ISO8601 UTC, 기한 `YYYY-MM-DD`, 금액 문자열(`"68,000원"`).
- 에러: `{ "error": { "code": "...", "message": "...", "details": {} } }`

### 1-2. ReadResult (POST /analyses 응답) — 스키마 고정, 신규 필드 없음

```jsonc
{
    "id": "string",
    "status": "done | processing | failed",
    "output_mode": "easy_korean | native",
    "lang": "ko | en | km | vi | ne | zh | th | id",
    "doc_type": "고지서 | 안내문 | 계약서 | 메시지 | 공지 | null",
    "summary_one_line": "string?",
    "original_text": "string?",
    "explained_text": "string?",
    "consequence": "string?",
    "cards": { "what": "s?", "when": "s?", "where": "s?", "amount": "s?", "deadline": "YYYY-MM-DD?" },
    "risks": [
        {
            "id": "s",
            "level": "green|yellow|red",
            "type": "s",
            "message": "짧은 한 줄",
            "detail": "string? (2~4문장)",
            "steps": ["string", "..."],
        },
    ],
    "actions": [{ "id": "s", "text": "s", "priority": 1, "is_done": false }],
    "reply_drafts": [{ "id": "s", "korean": "s", "note_in_lang": "s" }], // native만
    "created_at": "ISO8601",
}
```

> `risks[].detail`, `risks[].steps`는 위험 상세 화면용. 없으면 프론트가 `message`만 표시(폴백 있음). **스미싱 응답은 detail/steps를 채워주면 프론트 위험 상세가 바로 완성된다.**

### 1-3. HelpContactsResponse (GET /help-contacts 응답)

```jsonc
{ "items": [{ "id": "s", "title": "s", "org": "s", "phone": "s", "category": "s", "emergency": false }] }
```

## 2. BE-1 — `POST /v1/analyses` 스미싱/사기 판단 품질

### 위험 신호 (이 중 하나라도 있으면 사기/스미싱 위험)

| 신호          | 예시                                      |
| ------------- | ----------------------------------------- |
| 기관 사칭     | 국민건강보험·국세청·경찰·법원·택배사·은행 |
| 낯선 URL      | 비공식 도메인, 단축 URL, 오타 도메인      |
| 압박 문구     | 오늘까지·즉시 납부·압류·고발·계정 정지    |
| 금전 요구     | 미납금·수수료·과태료·보증금               |
| 개인정보 요구 | 주민번호·계좌·카드·인증번호·비밀번호      |
| 앱 설치 유도  | 보안 앱·확인 앱 설치                      |

### 프롬프트 추가 규칙 (시스템 프롬프트에 삽입)

```text
스미싱/피싱/사기 의심 문자는 별도 위험으로 판단한다.
공공기관·금융기관·택배사를 사칭하면서 낯선 URL, 단축/오타 도메인,
앱 설치 유도, 개인정보/금전 입력 유도, "오늘까지·즉시·압류·계정 정지"
같은 압박 문구가 있으면 red 또는(애매하면) yellow 위험으로 표시한다.

확실하지 않으면 "사기입니다" 단정 대신 "사기/스미싱이 의심돼요. 공식 채널로 확인하세요".

actions에는 관련되는 것을 포함:
- 문자 안의 링크를 누르지 않기
- 공식 앱/대표번호/공식 홈페이지로 직접 확인하기
- 118에 상담하기
- 이미 눌렀다면 비밀번호/카드/계좌 정보를 입력하지 않기
- (돈을 이미 보냈다면) 112 신고와 은행 지급정지를 안내

금액·날짜는 본문에 있는 그대로만 추출한다. 없는 사실은 만들지 않는다.
```

### 응답 필드 요구

| 필드               | 요구                                                    |
| ------------------ | ------------------------------------------------------- |
| `doc_type`         | `메시지`                                                |
| `summary_one_line` | "사기/스미싱일 수 있다"가 한눈에                        |
| `risks[].level`    | 명확한 사칭/URL/압박 → `red`, 애매 → `yellow`           |
| `risks[].type`     | 표준 `사기` (더 구체 원하면 `스미싱` 허용)              |
| `risks[].message`  | 링크 미클릭 + 공식 채널 확인                            |
| `risks[].detail`   | 왜 위험한지 2~4문장                                     |
| `risks[].steps`    | 링크 미클릭 / 공식 확인 / 118 상담 등 2~4개             |
| `cards.amount`     | 본문 금액 정확 추출, `cards.deadline` "오늘"만이면 null |
| `reply_drafts`     | easy_korean이면 `[]`                                    |

### 위험 타입 표준

`사기`(스미싱/피싱/사칭) · `임금`(급여/일당/부당공제) · `계약`(근로계약/숙소비/공제) · `기한`(납부/마감) · `과태료`(벌금) · `기타`. 발표 안정성 위해 사기성은 가급적 `사기`로 통일. varchar + 서버검증 권장(DB enum으로 배포 막지 말 것).

## 3. BE-2 — `GET /v1/help-contacts` `risk_type` + 연락처 확장

### 요청

```http
GET /v1/help-contacts?lang=ko&doc_type=메시지&risk_type=사기
```

| 파라미터    | 필수 | 값                                                         |
| ----------- | ---- | ---------------------------------------------------------- |
| `lang`      | no   | `ko`, `en` 등                                              |
| `doc_type`  | no   | `메시지`, `고지서`, `계약` 등                              |
| `risk_type` | no   | `사기`, `스미싱`, `임금`, `계약`, `과태료`, `기한`, `기타` |

### 연락처 기본 데이터 (static config — **★=이번에 추가**)

| id                 | title                    | org              | phone     | category | emergency |
| ------------------ | ------------------------ | ---------------- | --------- | -------- | --------- |
| `kisa_118`         | 스미싱·해킹 상담         | KISA/보호나라    | 118       | 사기     | false     |
| `police_112`       | 긴급 · 범죄 신고         | 경찰             | 112       | 긴급     | true      |
| `moel_1350`        | 임금·노동 상담           | 고용노동부       | 1350      | 임금     | false     |
| `immigration_1345` | 외국인종합안내센터       | 법무부           | 1345      | 외국인   | false     |
| `nhis_1577_1000`   | 건강보험 문의            | 국민건강보험공단 | 1577-1000 | 건강보험 | false     |
| ★ `fss_1332`       | 보이스피싱·지급정지 상담 | 금융감독원       | 1332      | 금융     | false     |
| ★ `legal_aid_132`  | 무료 법률상담            | 대한법률구조공단 | 132       | 법률     | false     |

> ★ 2개는 스미싱 **피해대응(벌써 당했다면)** CTA용. 금전 피해 시 112·은행 지급정지 → 이후 1332(지급정지·환급 상담)·132(무료 법률상담).

### 정렬 규칙 (risk_type별 1순위부터)

| risk_type                           | 반환 순서                                             |
| ----------------------------------- | ----------------------------------------------------- |
| `사기`,`스미싱`,`피싱`,`보이스피싱` | `kisa_118`, `police_112`, `fss_1332`, `legal_aid_132` |
| `임금`,`급여`,`일당`,`부당공제`     | `moel_1350`, `immigration_1345`                       |
| `계약`,`숙소비`,`공제`              | `moel_1350`, `immigration_1345`                       |
| `건강보험`                          | `nhis_1577_1000`, `kisa_118`                          |
| `과태료`,`기한`                     | 일반 목록(긴급 아님)                                  |
| 기타/없음                           | 기본 목록                                             |

### fallback

| 상황                        | 응답                                        |
| --------------------------- | ------------------------------------------- |
| `risk_type` 없음/알 수 없음 | 200 + 기본 목록                             |
| `lang` 미지원               | 한국어 기본 목록                            |
| phone 없는 항목             | 반환 제외 or phone null(프론트가 버튼 숨김) |

> 이 API는 **가능하면 실패하지 않는다**. static config면 500 확률↓.

## 5. DB / 보안 / 에러

- **DB:** 변경 필수 아님. `risks.type`이 `사기`/`스미싱` 저장 가능해야 함. `risks.detail`/`steps` 있으면 저장/반환. help-contacts는 static config 허용.
- **보안:** 서버는 URL 방문/실행 금지(텍스트 패턴만). 원문 전체 INFO 로그 금지, 주민번호/계좌/인증번호 마스킹. "사기 확정" 대신 "사기/스미싱 의심". 118/1350/132/1332는 상담·확인 안내로만(자동 신고 아님).
- **에러(`POST /analyses`):** 400 `INVALID_IMAGE` / 413 `PAYLOAD_TOO_LARGE` / 422 `AI_PARSE_FAILED` / 502 `AI_PROVIDER_ERROR` / 429 `RATE_LIMITED`.

## 6. 테스트

### Unit

- `risk_type=사기` → 1순위 `kisa_118`, 목록에 `police_112`·`fss_1332`·`legal_aid_132` 포함.
- `risk_type=스미싱` → 1순위 `kisa_118`.
- `risk_type=임금` → 1순위 `moel_1350`.
- `risk_type=계약` → `moel_1350`,`immigration_1345` 포함.
- 알 수 없는 `risk_type` → 기본 목록.
- phone 없는 항목 있어도 직렬화 실패 없음.

### Integration

- `GET /help-contacts?risk_type=사기` 200, `?risk_type=임금` 200.
- `POST /analyses` 스미싱 이미지 → `risks[].type ∈ {사기, 스미싱}`, `level ∈ {red, yellow}`.
- 같은 응답 `actions[]`에 링크 미클릭/공식 확인/118 중 최소 2개.

### Prompt/Eval Fixture (고정)

1. 공공기관 사칭 + 낯선 URL + 압박 → red.
2. 택배 사칭 + 앱 설치 유도 → red/yellow.
3. 실제 공공요금 고지서(공식) → 사기로 단정 안 함.
4. 임금 삭감 문자 → 임금 위험, 1350.

## 8. 프론트와 맞출 계약 (이것만 지키면 병렬 OK)

```text
POST /analyses → ReadResult (스키마 고정, 신규 필드 없음)
risks[].type contains 사기/스미싱 → help-contacts 118 우선
risks[].type contains 임금/계약/부당공제 → 1350 우선
help-contacts 첫 item = 가장 추천 CTA
help-contacts 사기 목록에 112·1332·132 포함 (프론트가 '벌써 당했다면' 2단 구성)
risks[].detail/steps = 위험 상세 본문/행동
actions[] = 결과 할 일 체크
text_scale = /profile 기존 필드 (E3가 재사용, 신규 아님)
```
