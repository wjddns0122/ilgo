## 1. 소비하는 API 계약 (백엔드와 동일 — 이 계약만 믿고 병렬 진행)

### 1-1. 분석 — POST /analyses
요청:
```json
{ "image_base64": "<BASE64>", "media_type": "image/jpeg", "output_mode": "easy_korean", "lang": "ko", "save": true }
```
응답 = `ReadResult` (스키마 고정, 신규 필드 없음):
```jsonc
{
  "id":"s","status":"done|processing|failed","output_mode":"easy_korean|native","lang":"ko|en|...",
  "doc_type":"메시지|고지서|...|null","summary_one_line":"s?","original_text":"s?","explained_text":"s?","consequence":"s?",
  "cards":{"what":"s?","when":"s?","where":"s?","amount":"s?","deadline":"YYYY-MM-DD?"},
  "risks":[{"id":"s","level":"green|yellow|red","type":"s","message":"s","detail":"s?","steps":["s"]}],
  "actions":[{"id":"s","text":"s","priority":1,"is_done":false}],
  "reply_drafts":[{"id":"s","korean":"s","note_in_lang":"s"}],
  "created_at":"ISO8601"
}
```
- 판정 로직(구현됨): `status=="failed"`→흐림, `doc_type` 불명→문서아님, 그 외→결과.
- 라이브 실패 폴백(구현됨): 실패 시 번들 예시 결과 + "예시 결과예요" 배지.

### 1-2. 도움처 — GET /help-contacts (risk_type 추가)
```http
GET /help-contacts?lang={profile.lang}&doc_type={analysis.doc_type}&risk_type={risk.type}
```
응답:
```json
{ "items": [ { "id":"kisa_118","title":"스미싱·해킹 상담","org":"KISA/보호나라","phone":"118","category":"사기","emergency":false } ] }
```
Retrofit 시그니처에 `risk_type` 추가 필요:
```dart
@GET('/help-contacts')
Future<HelpContactsResponse> helpContacts(
  @Query('lang') String? lang,
  @Query('doc_type') String? docType,
  @Query('risk_type') String? riskType,
);
```
연락처 데이터(백엔드 제공, 폴백도 동일하게 로컬 보유):
`kisa_118`(118) · `police_112`(112) · `moel_1350`(1350) · `immigration_1345`(1345) · `nhis_1577_1000`(1577-1000) · **`fss_1332`(1332 금감원)** · **`legal_aid_132`(132 법률구조)**.

### 1-3. 글씨크기 — /profile `text_scale` (기존 필드 재사용, E3)
`text_scale`: `0.9 / 1.0 / 1.15 / 1.3` (작게/보통/크게/아주크게). 현재 `shared_preferences` 로컬. **신규 필드 아님.**

## 2. FE-1 (E2) — 스미싱 피해대응 CTA 2단 + 도움요청 공유

### 2-1. 위험 상세 CTA를 2단으로 (`RiskDetailView` 하단)
```text
도움받을 곳  — 혼자 판단하지 않아도 돼요.

① 지금 확인
   [118 전화] KISA/보호나라 · 스미싱·해킹 상담
   [1350 전화] (임금/계약 위험일 때)
   [도움받을 곳 전체 보기]

② 벌써 당했다면            ← 접힘/보조. 탭하면 펼쳐짐(자동 아님)
   또바기(존댓말): "돈을 보내셨으면 지금 바로 112에 신고하고,
                  보낸 은행에 '지급정지'를 요청해요. 돌려받을 수 있는 공식 절차가 있어요."
   [112 즉시 신고]  [1332 지급정지·상담]  [132 무료 법률상담]
```
- CTA 소스: `GET /help-contacts?risk_type=...` 결과. **API 실패 시 로컬 폴백 매핑**(2-2)으로 동일하게 렌더.
- `②`는 사기/스미싱 `red` 위험에서 항상 노출되되 **접혀 있고**, 탭 시 피해대응 안내 + 또바기 존댓말 동행이 펼쳐진다.
- 톤 규칙: **"돌려받을 수 있는 공식 절차가 있어요"(안내)** — "보상 보장" 금지.

### 2-2. 로컬 폴백 매핑 (API 실패해도 CTA 보이게)
| `risk.type` 포함 | ① 지금 확인 | ② 벌써 당했다면 |
|---|---|---|
| `사기`·`스미싱`·`피싱`·`보이스피싱` | 118 | 112, 1332, 132 |
| `임금`·`급여`·`일당`·`부당공제` | 1350 | 1350, 1345 (임금은 대지급금 안내) |
| `계약`·`숙소비`·`공제` | 1350 | 1345 |
| `건강보험` | 1577-1000 | 도움받을 곳 전체 |
| `과태료`·`기한`·기타 | 도움받을 곳 전체 | — |

### 2-3. 전화 동작
- CTA 탭 → `tel:` 로 전화 앱 열기(바로 통화 아님, OS 확인). 기존 도움받을 곳 다이얼 코드 재사용.
- 실패 시 스낵바: `전화 앱을 열 수 없어요. 번호를 직접 눌러주세요: 118`.

### 2-4. 도움 요청 공유 (`share_plus`)
공유 버튼 → 선택(또는 기본 공유 문구 교체):
```text
[읽고] 제가 받은 글을 쉽게 정리했어요.

요약: {summary_one_line}
위험: {risk_label} · {risk_type} — {risk_message}
해야 할 일:
1. {action_1}
2. {action_2}

도움받을 곳: {contact_title} {phone}

혹시 같이 확인해줄 수 있나요?
```
- 위험 = red>yellow>green 최상위 1개. 할 일 = priority 낮은 순 1~2개. 연락처 = risk.type 추천 1순위.
- 피해 케이스(사기 red + ② 펼침)면 도움처에 112/1332 포함.
- **개인정보:** `original_text` 전체·이미지 공유 금지. 계좌/주민번호/전화번호 같은 긴 숫자 직접 삽입 금지. `cards.amount`/`deadline` 수준만.

## 3. FE-2 (E3) — 온보딩 글씨크기 실시간 미리보기

- 위치: 온보딩 플로우에 "글씨 크기" 단계 추가(모드·언어 선택 근처). 한국어(쉬운한국어) 모드에서 특히 부각.
- UI: 슬라이더(작게 0.9 / 보통 1.0 / 크게 1.15 / 아주크게 1.3) + **미리보기 문장이 실시간 리사이즈**.
  ```text
  미리보기: "건강보험료 68,000원, 7월 15일까지 내세요."   ← 슬라이더 따라 즉시 커짐/작아짐
  ```
- 저장: 기존 `text_scale` 키(`shared_preferences`)에 저장 → 앱 전체(결과·설정 포함) 반영. **기존 설정 글씨크기와 동일 키 사용**(중복 저장 금지).
- 또바기 1줄(반말): "글씨 잘 보여? 크게도 할 수 있어. 슬라이더를 움직여봐."
- 기존 text size 조정 뷰(커밋 `efae0ed`)를 온보딩 단계로 승격 + 미리보기 문장 추가하는 방향(새 저장소 만들지 말 것).

## 4. FE-3 (E1) — 또바기 캐릭터

> 상태머신·화면별 대사 스크립트·에셋 목록은 `읽고_캐릭터_컴패니언_또바기_스펙_2026-07-02.md`. 여기선 통합 지점만.

### 4-1. 결과화면 레이아웃 (히어로 무대)
```text
[결과 화면]
 기존 TTS 낭독 블록 (ListenTtsView / 음성 낭독)   ← 그대로 유지
 ────────────────────────────────────────
 또바기 "쉬운 보기" 섹션 (신규, 아래에 얹기)
   [또바기 캐릭터]  볼 색 = 신호등(초/노/빨)
   큰 글씨 한 줄 요약({summary_one_line})
   [👆 탭하면 또바기가 읽어줘]   ← TTS 탭 재생(자동재생 아님)
```
- **① 들려주기(기존 TTS) + ② 쉽게 보여주기(또바기)** 이중 채널. 또바기는 기존 기능 **대체 안 함**, 아래에 얹는 레이어.

### 4-2. 위젯 & 컨트롤러
- `TobagiController`(GetX): 현재 `ReadResult` + 라우트 이벤트 구독 → 상태(idle/thinking/reading/warn/safe/cheer) 산출.
- 배치: 결과화면 하단 섹션(오버레이 아님) + 온보딩/로딩은 라이트 등장.
- 애니: Rive 1순위 / Lottie / 이미지 상태 5장(폴백). 볼 색만 신호등 연동.

### 4-3. 대사 = 스크립트 결정형 (라이브 챗 아님)
`buildTobagiLine(context, readResult) → {state, text, register(반말|존댓말), speak}`:
- result(탭) → reading·반말·`summary_one_line`·speak=true
- topRisk red → warn·**존댓말**·경고+`risks[0].message` / yellow → caution·존댓말 / green∅ → safe·반말
- actionDone → cheer·반말 / smishingVictim → 존댓말(§2-1 ② 탭 시)
- **말투 규칙:** 위험·피해 안내 = 존댓말, 그 외 = 반말. **영어(lang=en) 출력은 반말/존댓말 구분 없이 따뜻한 단일 톤.**

### 4-4. 음성 = 탭하면 재생
- 결과화면 기본 **탭 재생**(자동재생 아님). 설정 토글 "결과 열면 자동 읽기"(기본 off).
- 기존 `TtsService` 재사용. 낭독은 짧게(요약·경고문). 원문 전체는 기존 `ListenTtsView`.

### 4-5. 접근성 & 폴백 (가드레일)
- **G3:** 항상 스킵/음소거. 또바기 롱프레스 = 음소거. 대사는 **자막**도 표시(청각 의존 금지). "애니 줄이기" 시 정적.
- **G4:** 에셋 로드 실패 → 위젯 **조용히 숨김**(빈 SizedBox), 앱 100% 동작. 또바기는 의존성 아님.
- 볼 신호등 색은 텍스트 위험 배지와 **중복 정보**(색만으로 전달 금지).
- 자막 글씨는 `text_scale`(E3) 따름.

## 5. 데이터 매핑 (참고)
- 위험 정렬: red=3 > yellow=2 > green=1, unknown=2(위험 숨기지 않음).
- 위험 라벨: red=빨강, yellow=노랑, green=초록, 기타=주의.
- 위험 타입 매칭: 완전일치 대신 **포함검색 폴백**(백엔드가 `스미싱`/`사기` 섞어 보내도 흡수).

## 6. 테스트

### Unit
- `buildTobagiLine`이 red/yellow/green/∅ → 맞는 state·register·text.
- 말투: 위험/피해=존댓말, 그 외=반말. lang=en=단일 톤.
- risk.type `사기`/`스미싱`→118, `임금`/`계약`→1350, 알 수 없음→폴백.
- 도움요청 공유 텍스트에 `original_text` 전체 미포함.

### Widget / Flow
- 홈 예시 바텀시트 스미싱 옵션 표시.
- 위험 상세 CTA 2단(①지금확인 / ②벌써당했다면 접힘→탭 펼침).
- 결과화면 또바기 "쉬운 보기" 섹션 렌더, **탭 전 무음**, 탭 시 낭독.
- 온보딩 글씨크기 슬라이더 → 미리보기 실시간 리사이즈 + 저장값 앱 반영.
- 또바기 에셋 없음 → 위젯 숨김, 크래시 없음.

### Manual
- 실서버 스미싱 예시 분석 성공 / 네트워크 오류 시 크래시 없음.
- iOS/Android `tel:` 동작(118/112/1350/1332/132).
- 공유 시트 정상.
- 실기기 TTS ko/en 낭독(또바기 탭).
