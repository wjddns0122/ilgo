## 1. 문서 목적과 범위

- **목적:** "읽어드림" MVP를 코드로 옮길 수 있도록 기능을 ID 단위로 정의하고, 각 기능의 입력·처리·출력·예외·수용 기준을 명시한다.
- **범위:** 해커톤 데모에 필요한 MVP 기능. 계정/서버/결제/푸시 등은 범위 외(§13).
- **읽는 법:** §5 기능 목록에서 우선순위를 보고, §6에서 상세를 본다. 데이터/API 계약은 §7~8.

### 우선순위 정의
| 등급 | 의미 |
|---|---|
| **MVP** | 데모 핵심 경로. 없으면 발표 불가. 반드시 구현. |
| **P1** | 완성도·플랫폼 증명. 시간 되면 구현. |
| **P2** | 있으면 좋음. 시간 부족 시 첫 번째 컷. |

## 2. 제품 개요

- **한 줄:** 어려운 한국어 글(종이 우편물 또는 화면 메시지)을 사진 한 장으로 찍으면, 사용자가 읽을 수 있는 말(쉬운 한국어 또는 모국어)로 풀어주고 위험을 짚고 할 일까지 정리하는 앱.
- **두 사용자 모드:** ① 쉬운 한국어(발달장애·저문해·어르신) ② 모국어(이주민).
- **핵심 엔진:** 입력 이미지 → **Claude 멀티모달 단일 호출** → 구조화 JSON → 화면 카드.

## 3. 용어 정의

| 용어 | 정의 |
|---|---|
| 이해 엔진 | 이미지 → Claude 1회 호출 → `ReadResult` JSON을 만드는 핵심 파이프라인 |
| 출력 모드 | `easy_korean`(쉬운 한국어) 또는 `native`(모국어). 출력 언어/톤을 결정 |
| `ReadResult` | Claude 응답 JSON을 매핑한 앱 내부 모델(§7.1) |
| 위험 신호등 | green/yellow/red 3단계 위험 표시 |
| 폴백 | 라이브 호출 실패 시 미리 저장한 분석 결과를 보여주는 데모 안전망 |

## 4. 시스템 구성 / 데이터 흐름

```
┌─────────────────────────── Flutter App ───────────────────────────┐
│                                                                    │
│  [온보딩]→ output_mode/lang 저장(SharedPreferences)                  │
│                                                                    │
│  [홈] ─촬영/선택→ image_picker ─→ 이미지 압축(1280px) ─→ base64       │
│                                            │                        │
│                                            ▼                        │
│                                   ┌──────────────────┐              │
│                                   │ ClaudeService    │  dio POST     │
│                                   │ (이해 엔진)        │──────────────┼──▶ Anthropic
│                                   └──────────────────┘              │   Messages API
│                                            │  JSON                  │   (멀티모달)
│                                            ▼                        │◀──┘ JSON
│                                   jsonDecode → ReadResult           │
│                                            │                        │
│              ┌─────────────────────────────┼───────────────────┐    │
│              ▼              ▼               ▼          ▼         ▼    │
│         결과 메인       원문↔풀이 토글    핵심 카드   위험 신호등  액션  │
│         (요약+신호등)                              할일/캘린더/답장/TTS │
│                                            │                        │
│                                            ▼                        │
│                                   보관함(Hive/SharedPreferences)      │
└────────────────────────────────────────────────────────────────────┘
```

**데이터 흐름 4경로 (해피 + 섀도):**
1. 정상: 이미지 → JSON → 카드 렌더.
2. 입력 없음(nil): 촬영/선택 취소 → 홈 유지, 진행 차단.
3. 빈/저품질 입력: 인식 불가 → "다시 찍어주세요" 안내.
4. 업스트림 오류: 네트워크/타임아웃/JSON 깨짐 → 폴백 또는 재시도(§6 SYS).

## 6. 상세 기능 명세

> 형식: **설명 / 선행조건 / 입력 / 처리 / 출력 / 예외·엣지 / 수용기준(AC)**

### ONB-01 · 온보딩·출력 모드 선택 [MVP]
- **설명:** 최초 실행 시 출력 모드를 선택한다. "쉬운 한국어로 볼까요? / 내 언어로 볼까요?"
- **선행조건:** 앱 최초 실행(저장된 모드 없음).
- **입력:** 모드 선택 탭(easy_korean / native).
- **처리:** 선택값을 `SharedPreferences`에 `output_mode` 저장. native면 ONB-02로.
- **출력:** 홈(IN) 화면 진입.
- **예외·엣지:** 이미 모드 저장됨 → 온보딩 건너뛰고 홈 직행. 설정(SET-01)에서 변경 가능.
- **AC:**
  - [ ] 최초 1회만 노출, 재실행 시 홈 직행.
  - [ ] 선택한 모드가 이후 모든 분석 호출의 `output_mode` 파라미터에 반영된다.

### ONB-02 · 모국어 언어 선택 [P1]
- **설명:** native 모드에서 모국어를 고른다.
- **입력:** 언어 목록에서 선택(데모: 크메르어 기본 + 베트남/네팔/영어 더미 가능).
- **처리:** `lang`(ISO 코드) 저장.
- **출력:** 홈 진입.
- **예외·엣지:** 미선택 → 다음 비활성. 지원 외 언어는 목록 미노출.
- **AC:**
  - [ ] 선택 `lang`이 호출 파라미터·UI 글리프 폰트에 반영.
  - [ ] 데모 최소 1개 언어(크메르어) 완전 동작.

### IN-01 · 카메라 촬영 입력 [MVP]
- **설명:** 종이 우편물을 카메라로 촬영.
- **입력:** "📷 종이 찍기" 버튼 탭 → 카메라.
- **처리:** `image_picker(source: camera)`로 이미지 획득 → IN-03.
- **출력:** 분석 로딩(ENG-03) 진입.
- **예외·엣지:** 카메라 권한 거부 → 권한 안내 + 설정 이동 버튼. 촬영 취소 → 홈 유지.
- **AC:**
  - [ ] 촬영 성공 시 즉시 분석 플로우 시작.
  - [ ] 권한 거부가 크래시 없이 안내로 처리된다.

### IN-02 · 갤러리/캡처 이미지 선택 [MVP]
- **설명:** 휴대폰에 저장된 화면 캡처(카톡/문자/공지)나 사진 선택.
- **입력:** "🖼️ 화면 캡처 넣기" 버튼 탭 → 갤러리.
- **처리:** `image_picker(source: gallery)` → IN-03.
- **출력:** 분석 로딩 진입.
- **예외·엣지:** 선택 취소 → 홈 유지. 비이미지/손상 파일 → "이미지를 읽을 수 없어요" 안내.
- **AC:**
  - [ ] 캡처 이미지로도 IN-01과 동일 경로로 분석된다.

### IN-03 · 이미지 압축·base64 인코딩 [MVP]
- **설명:** 업로드 속도·비용 안정화를 위해 리사이즈 후 인코딩.
- **입력:** 원본 이미지 파일.
- **처리:** `flutter_image_compress`로 가로 최대 1280px·품질 ~80% → `base64Encode`. MIME(jpeg/png) 판별.
- **출력:** base64 문자열 + media type → ENG-01.
- **예외·엣지:** 인코딩 실패 → 오류 토스트 + 홈 복귀. 초대형 이미지도 리사이즈 후 처리.
- **AC:**
  - [ ] 인코딩 결과가 API 이미지 블록 규격(base64 + media_type)에 맞는다.

### ENG-01 · Claude 분석 호출 (이해 엔진) [MVP] ★핵심
- **설명:** base64 이미지 + 시스템 프롬프트(+모드/언어)로 **단일** Messages API 호출.
- **선행조건:** base64 이미지, `output_mode`/`lang`, API 키(`--dart-define`).
- **입력:** §8 요청 바디.
- **처리:** `dio.post`(timeout 30s). 헤더 `x-api-key`, `anthropic-version`. assistant prefill `{`로 JSON 강제. `temperature 0.2`, `max_tokens ~1200`.
- **출력:** 원시 JSON 문자열 → ENG-02.
- **예외·엣지:** 타임아웃/네트워크/4xx·5xx/레이트리밋 → SYS-01. 호출 1회 원칙(추가 라운드트립 없음).
- **AC:**
  - [ ] 1회 호출로 요약·풀이·카드·위험·액션·(native)답장 전부 반환.
  - [ ] `output_mode=native` 시 응답 언어가 `lang`과 일치.

### ENG-02 · JSON 파싱 → ReadResult [MVP]
- **설명:** 응답을 `ReadResult`(§7.1)로 디코드.
- **처리:** `jsonDecode` → 모델 매핑. 누락 필드는 null/빈 배열 기본값.
- **출력:** `ReadResult` 객체.
- **예외·엣지:** JSON 깨짐/스키마 불일치 → 재시도 1회(ENG-01) → 실패 시 `explained_text`만이라도 추출 시도 → 그래도 실패 시 SYS-01 오류 화면.
- **AC:**
  - [ ] 필수 필드 누락에도 크래시 없이 부분 렌더.
  - [ ] 깨진 응답이 재시도 후 graceful 처리된다.

### ENG-03 · 분석 상태 관리 [MVP]
- **설명:** `idle → loading → success | error` 상태머신(provider).
- **출력:** 상태별 화면 전환(로딩/결과/오류).
- **예외·엣지:** 분석 중 뒤로가기/화면 이탈 → 진행 취소 또는 결과 폐기(레이스 방지). 로딩 중 중복 요청 차단(버튼 비활성).
- **AC:**
  - [ ] 동일 입력 더블탭이 중복 호출을 만들지 않는다.
  - [ ] 분석 중 이탈 시 stale 결과가 다음 화면에 끼어들지 않는다.

```
상태 다이어그램 (분석)
 idle ──촬영/선택──▶ loading ──성공──▶ success ──다시─▶ idle
   ▲                  │  │
   │                  │  └─실패──▶ error ──재시도─▶ loading
   └──────취소/뒤로◀───┘            └─폴백─▶ success(캐시)
```

### RES-01 · 핵심 한 문장 요약 [MVP]
- **설명:** 가장 먼저 보이는 단 하나의 답. 큰 글씨.
- **입력:** `ReadResult.summary_one_line`.
- **출력:** 결과 메인 상단 큰 카드 + 금액/기한 뱃지(있으면).
- **예외·엣지:** 요약 null → "핵심을 찾지 못했어요. 원문을 확인해 주세요" + 원문 표시.
- **AC:** [ ] 요약이 화면 최상단, 한 화면에 한 문장으로 보인다.

### RES-02 · 원문 ↔ 풀이 토글 [MVP] ★데모 핵심
- **설명:** 어려운 원문과 쉬운말/모국어 풀이를 토글로 비교.
- **입력:** `original_text`, `explained_text`.
- **처리:** 상단 세그먼트 토글로 두 텍스트 전환. easy 모드=쉬운 한국어, native=모국어.
- **출력:** 토글 화면.
- **예외·엣지:** 한쪽 텍스트 null → 가능한 쪽만 표시, 토글 비활성.
- **AC:**
  - [ ] 토글 1탭으로 원문↔풀이가 즉시 전환된다(AI 변환의 before/after가 눈에 보임).

### RES-03 · 핵심 추출 카드 [MVP]
- **설명:** 무엇/언제까지/어디서/얼마를 구조화 카드로.
- **입력:** `ReadResult.cards{what,when,where,amount,deadline}`.
- **출력:** 값이 있는 항목만 카드 칩으로 렌더.
- **예외·엣지:** 전체 null(추출 없음) → 카드 영역 숨김(빈 상태).
- **AC:** [ ] null 항목은 빈 칸 없이 숨겨진다.

### RES-04 · 위험 신호등 + 상세 [MVP]
- **설명:** green/yellow/red 신호등 배너 + 탭 시 사유.
- **입력:** `ReadResult.risks[{level,type,message}]`.
- **처리:** 최고 위험 레벨로 배너 색 결정. 탭 → 위험 목록·사유(모드 언어).
- **출력:** 색상 배너 + 상세.
- **예외·엣지:** risks 빈 배열 → green "특별한 위험은 안 보여요"(단정 금지 톤). 다중 위험 → 빨강 우선 노출.
- **AC:**
  - [ ] red가 있으면 배너가 즉시 빨강.
  - [ ] 위험 문구가 '주의/확인' 톤(단정 회피).

### RES-05 · 할 일 체크리스트 + '안 하면?' [MVP]
- **설명:** 따라 하면 되는 행동 1·2·3 + 미이행 결과.
- **입력:** `actions[{text,priority}]`, `consequence`.
- **출력:** 체크 가능한 리스트 + 결과 설명 박스.
- **예외·엣지:** actions 빈 배열 → "지금 할 일은 없어요". consequence null → 박스 숨김.
- **AC:** [ ] priority 순 정렬, 체크 토글 동작.

### ACT-01 · 캘린더에 기한 추가 [MVP]
- **설명:** `deadline`을 기기 캘린더 이벤트로 등록.
- **입력:** `cards.deadline`(YYYY-MM-DD), 요약 텍스트.
- **처리:** `add_2_calendar`로 이벤트(제목=요약, 날짜=deadline) 추가.
- **출력:** 네이티브 캘린더 추가 시트.
- **예외·엣지:** deadline null → 버튼 비활성. 권한 거부 → 안내.
- **AC:** [ ] 버튼 탭 시 실제 캘린더에 해당 날짜 이벤트가 생성된다(데모 클라이맥스).

### ACT-02 · 전화 걸기 [P2]
- **설명:** 담당 기관 번호로 전화.
- **입력:** 문서에서 추출/고정된 전화번호.
- **처리:** `url_launcher`로 `tel:` 실행.
- **예외·엣지:** 번호 없음 → 버튼 숨김.
- **AC:** [ ] 번호 있을 때만 노출, 탭 시 전화 앱 실행.

### ACT-03 · 한국어 답장 생성·복사 [P1] (native 전용)
- **설명:** 상황에 맞는 정중한 한국어 답장 후보 제공.
- **입력:** `reply_drafts[{korean,note_in_lang}]`(ENG-01 응답).
- **처리:** 후보 2~3개 카드 표시(한국어 + 모국어 해설). '복사' → 클립보드.
- **출력:** 답장 화면.
- **예외·엣지:** native 아니면 화면 미노출. reply_drafts 빈 배열 → "답장이 필요 없는 메시지예요".
- **AC:**
  - [ ] native 모드에서 답장 후보가 보이고, 복사 후 카톡에 붙여넣기 가능.

### ACT-04 · 핵심 한 줄 음성 낭독 [P1] (easy_korean 전용)
- **설명:** 글을 못 읽어도 귀로 듣게.
- **입력:** `summary_one_line`.
- **처리:** `flutter_tts`(ko-KR)로 낭독. 재생/정지 버튼.
- **예외·엣지:** TTS 엔진 없음 → 버튼 숨김(크래시 금지).
- **AC:** [ ] 버튼 탭 시 한국어로 요약을 읽어준다.

### LIB-01 · 보관함 [P2]
- **설명:** 분석한 글 목록 저장·재열람.
- **입력:** `ReadResult` + 타임스탬프.
- **처리:** `hive`/`SharedPreferences`에 JSON 직렬화 저장. 목록은 날짜·위험 색·기한순.
- **예외·엣지:** 저장 실패 → 기능 무시(분석 자체엔 영향 없음).
- **AC:** [ ] 저장된 항목을 다시 열면 동일 결과가 렌더된다.

### SET-01 · 설정(모드/언어 변경) [P1]
- **설명:** 출력 모드·모국어를 나중에 변경.
- **처리:** 변경값 저장, 다음 분석부터 적용.
- **AC:** [ ] 변경 즉시 다음 호출 파라미터에 반영.

### SYS-01 · 오류·재시도·폴백 [MVP] ★데모 안전망
- **설명:** 라이브 실패 시에도 발표가 멈추지 않게.
- **처리/규칙:**
  - 네트워크/타임아웃 → "잠시 후 다시 시도" + 재시도 버튼.
  - 호출 실패/JSON 깨짐 → 자동 재시도 1회.
  - 그래도 실패 → **사전 분석 결과 캐시**(데모용 시드 입력)로 폴백 표시.
  - 인식 불가/저품질 → "다시 찍어주세요" 안내(저신뢰 시드).
- **예외·엣지:** 오프라인 → 캐시 폴백 우선. 레이트리밋(429) → 지수 백오프 1회.
- **AC:**
  - [ ] 라이브 실패가 빈 화면/크래시로 이어지지 않는다.
  - [ ] 사전 캐시 폴백이 0~1초 내 표시된다.

### SYS-02 · 접근성 [MVP]
- **설명:** 당사자(저문해·발달장애·고령) 사용성.
- **규칙:** 최소 본문 18sp+·핵심 24sp+, 고대비 색, 한 화면 한 정보, 이모지·존댓말, 터치 타깃 48dp+.
- **AC:** [ ] 결과 메인이 큰 글씨·고대비로 한눈에 읽힌다.

## 7. 데이터 모델

### 7.1 ReadResult (Claude 응답 매핑)
```jsonc
{
  "doc_type": "고지서",                 // 고지서|공지|메시지|계약|기타
  "summary_one_line": "8월 15일까지 건강보험료 6만 8천 원을 내야 해요",
  "original_text": "…문서 원문(한국어)…",
  "explained_text": "…쉬운 한국어 또는 모국어 풀이…",
  "cards": {
    "what": "건강보험료 납부",
    "when": null,
    "where": null,
    "amount": "68,000원",
    "deadline": "2026-08-15"           // YYYY-MM-DD | null
  },
  "risks": [
    { "level": "yellow", "type": "기한", "message": "8월 15일이 지나면 연체료가 붙을 수 있어요" }
  ],
  "actions": [
    { "text": "은행 앱이나 지로로 6만 8천 원을 내세요", "priority": 1 }
  ],
  "consequence": "기한이 지나면 연체료가 더해질 수 있어요",
  "reply_drafts": [                     // native 모드에서만 채움
    { "korean": "네, 확인했습니다. 기한 안에 납부하겠습니다.", "note_in_lang": "…모국어 해설…" }
  ]
}
```
**필드 규칙:** 금액·날짜는 본문에서 정확 인용, 불확실하면 null(추측 금지). `level`은 확신 없으면 yellow. 없는 사실 생성 금지.

### 7.2 UserProfile (로컬)
```jsonc
{ "output_mode": "easy_korean", "lang": "ko" }   // native면 lang="km" 등
```

### 7.3 SavedItem (보관함, P2)
```jsonc
{ "id": "uuid", "created_at": "2026-06-30T07:10:00", "result": { /* ReadResult */ } }
```

## 8. API 명세 (Anthropic Messages)

### 8.1 요청
- **Endpoint:** `POST https://api.anthropic.com/v1/messages`
- **Headers:** `x-api-key: <KEY>`, `anthropic-version: 2023-06-01`, `content-type: application/json`
- **Body(요지):**
```jsonc
{
  "model": "claude-opus-4-8",        // 속도 우선 시 claude-sonnet-4-6
  "max_tokens": 1200,
  "temperature": 0.2,
  "system": "<§8.3 시스템 프롬프트>",
  "messages": [
    { "role": "user", "content": [
      { "type": "image", "source": { "type": "base64", "media_type": "image/jpeg", "data": "<BASE64>" } },
      { "type": "text", "text": "user_profile={\"output_mode\":\"easy_korean\",\"lang\":\"ko\"}. 위 이미지를 분석해 지정한 JSON으로만 답하세요." }
    ]},
    { "role": "assistant", "content": "{" }   // JSON 강제 prefill
  ]
}
```

### 8.2 응답 처리
- `content[0].text`(prefill 포함 시 `{`로 시작) → `jsonDecode` → ReadResult.
- 파싱 실패 → ENG-02 예외 경로.

### 8.3 시스템 프롬프트 (스케치)
> "당신은 한국어 글을 못 읽는 사람을 돕는 '읽어드림' 도우미입니다. 첨부 이미지는 한국 행정·금융 우편물 또는 메시지(카톡/문자/공지) 화면입니다. `user_profile.output_mode`가 `easy_korean`이면 모든 출력을 초등 2학년도 이해할 쉬운 한국어(짧은 문장·이모지·존댓말)로, `native`이면 `lang` 언어로 **직역이 아닌 의역**(줄임말·존댓말 뉘앙스 반영)으로 작성하세요. 다음 키를 가진 **JSON으로만** 답하세요: `doc_type, summary_one_line, original_text, explained_text, cards{what,when,where,amount,deadline}, risks[{level(green|yellow|red),type,message}], actions[{text,priority}], consequence, reply_drafts[{korean,note_in_lang}]`(reply_drafts는 native일 때만). 위험은 스미싱·임금차감·부당공제·과태료·신고기한 등 사용자가 손해 볼 신호이며 **확신이 없으면 red로 단정하지 말고 yellow**로. **금액·날짜는 본문에서 정확히 인용하고 불확실하면 null로 두며, 본문에 없는 사실은 만들지 마세요.** JSON 외 텍스트 금지."

## 12. 수용 기준 — 데모 성공 정의 (Definition of Done)

- [ ] **(쉬운 한국어)** 실제 종이 고지서 촬영 → 5~7초 내 핵심 한 문장 + 원문↔쉬운말 토글 + 캘린더 등록까지 라이브 동작.
- [ ] **(모국어)** 한국어 카톡 캡처 → 모국어 의역 + 위험 신호등(임금/기한) + 한국어 답장 복사 동작.
- [ ] 모드 전환이 같은 앱·같은 엔진으로 즉시 반영("같은 앱, 다른 사람" 증명).
- [ ] 라이브 실패 시 캐시 폴백이 작동해 발표가 멈추지 않는다.
- [ ] 잘못된 금액/날짜를 자신 있게 단정하지 않는다(불확실 시 null·원문 토글로 대조 가능).
