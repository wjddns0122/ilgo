> **대주제:** AI for Everyone — "누군가를 위한 AI를 설계하라"

## 0. 전략 한 장 요약

**우리만의 무기 = "지금 작동하는 AI".** 다른 팀은 Canva 클릭목업(`이렇게 될 거예요`)을 낸다. 우리는 심사위원 앞에서 실제 구동되는 Flutter 앱에 라이브 Claude 호출을 돌려 `지금 됩니다`를 보여준다. 이 한 순간이 **완성도(15)** 와 **AI 활용도(20)** 에서 격차를 만든다.

### 심사 기준 (100점)
| 항목 | 배점 | 우리 전략 |
|---|---|---|
| 사회적 가치·공감 | **30** | 구체적 페르소나 1명의 하루를 발표 첫 90초에 |
| 아이디어 창의성 | **25** | "번역/요약을 넘어 행동·자기보호까지" 가치사슬 확장 |
| AI 활용도 | **20** | **단 1회 Claude 멀티모달 호출**이 화면에서 눈에 보이게 |
| 결과물 완성도 | **15** | 실제 구동 Flutter 앱 (빈/에러/히스토리까지) |
| 발표 전달력 | **10** | 7분을 1개 시나리오로, 심사위원 즉석 라이브가 클라이맥스 |

### 핵심 인사이트 — 4개 중 3개가 같은 엔진
실버·다문화·배리어프리 우승작은 **"이미지/캡처 1장 → Claude 멀티모달 1회 호출 → 구조화 JSON → 카드 UI"** 라는 동일 엔진을 공유한다. → **엔진 1개 만들어 트랙별로 리스킨**하면 2일 안에 어느 것이든, 심지어 여러 개를 만들 수 있다. 돌봄(교대인계)만 입력이 **음성**이라 가장 차별적(받아쓰기가 아니라 추론).

## 2. 🥈 실버 트랙 — 찰칵 효도 (Chalkak Hyodo)
> "종이 한 장 찍으면 AI가 신호등으로 답해드려요. 초록·노랑·빨강."

**한 줄 정리:** 어르신이 우편물·문자를 카메라로 찍으면, 큰 글씨 3줄 요약 + 사기 신호등 판정 + 손주 톤 음성 안내를 주는 앱.

**문제 정의**
어르신은 문서의 '내용 이해'와 '진위 판단'에서 동시에 소외된다. 작은 글씨 + 행정용어 + 사기 구분 불가가 겹쳐 우편물·문자 한 장이 공포다. 65세 이상 스마트폰 사기 피해는 매년 늘고, 정보격차가 곧 사기 취약성으로 이어진다. 결국 매번 자식에게 전화로 "이거 뭐야?", "이거 사기야?"를 물어야 하는 의존이 일상이 된다.

**페르소나**
73세 김순자. 건강보험 고지서·관리비 안내문이 쌓이고 그 사이에 스미싱 의심 문자가 섞인다. 작은 글씨를 못 읽어 매번 자식에게 전화로 묻는다. 폰은 전화·카톡만 쓴다. 고통점: (1) 무슨 내용인지 모름, (2) 진짜인지 사기인지 모름, (3) 뭘 해야 하는지 모름, (4) 매번 묻기 미안함.

**HMW (How Might We)**
글씨가 안 보이고 사기 구분이 안 되는 어르신이, 종이/문자를 사진 한 장으로 찍어 '무슨 내용인지·진짜인지·뭘 해야 하는지'를 혼자서도 알게 하려면?

**가치 제안**
찰칵 효도는 어르신의 '읽기'와 '의심'을 동시에 대신해준다. 사진 한 장이면 큰 글씨 3줄 요약 + 사기 신호등 + 해야 할 일 한 가지를 음성으로 안내해, 자식에게 전화하지 않고도 안전하게 판단·행동하게 한다.

**MVP 기능** (2일 빌드 스코프)
| 기능 | 왜 필요 | Claude 사용 | Claude 역할 |
|---|---|---|---|
| 사진 1장 촬영/선택 | 어르신 행동(찍어서 묻기)을 그대로 입력으로 | ✗ | — |
| 큰 글씨 3줄 핵심 요약 | '읽을 수 없는 벽'을 즉시 허묾 | ✓ | 멀티모달 비전으로 문서 읽고 핵심 3줄 압축 |
| 사기 신호등 (초록/노랑/빨강) | 진위 판단을 대신 — 사회가치+창의의 핵심 | ✓ | 스미싱/피싱 패턴 추론해 risk_level + 이유 반환 |
| 해야 할 일 1가지 + 음성 안내 | 이해→행동, 글 못 읽어도 귀로 | ✓ | action 1건 생성 → flutter_tts로 손주 톤 낭독 |
| 기록 보관 (찍은 문서 목록) | 나중에 자식과 확인, 누적 가치 | ✗ | — |

**화면 구성** (5장+)
1. **홈 / 찍기** — 큰 카메라 버튼 하나 "사진 찍기". 아래 최근 기록 1~2건. 진입장벽 0.
2. **분석 중** — 찍은 사진 + "AI가 읽고 있어요" 애니메이션. 라이브 호출 가시화.
3. **결과 신호등 (핵심)** — 화면 상단 큰 신호등 색 배너 + 큰 글씨 3줄 요약. 데모 클라이맥스.
4. **자세히/왜 위험한가** — 빨강이면 "이래서 사기 같아요(예: 링크 클릭 유도, 기관 사칭)" 음성+텍스트.
5. **해야 할 일** — "○○에 전화하세요" 한 가지 + 전화 걸기 버튼.
6. **내 문서함** — 찍은 문서 카드 목록, 신호등 색 뱃지.

**Claude 연동 (구체)**
사진 1장 → base64 인코딩 → Anthropic Messages API에 image 블록 + 시스템 프롬프트로 **단 1회 호출** → 엄격한 JSON 반환 → 카드/신호등 렌더 → flutter_tts 낭독. 모델: `claude-opus-4-8` (정확도 우선) 또는 데모 속도용 `claude-sonnet-4-6`. temperature 0.2.

- 시스템 프롬프트 스케치:
  > "당신은 디지털이 낯선 한국 어르신을 돕는 도우미입니다. 첨부 이미지는 어르신이 받은 우편물/문자 캡처입니다. JSON으로만 답하세요: `summary3`(큰글씨용 핵심 3줄, 쉬운말·존댓말), `risk_level`(green|yellow|red), `risk_reason`(왜 그렇게 봤는지 쉬운말, 확신 없으면 yellow), `action`(어르신이 할 일 한 가지, 없으면 null), `amount`/`due_date`(있으면). **확신이 없으면 절대 green으로 단정하지 말고 yellow로.** 본문에 없는 사실은 만들지 마세요."
- JSON 스키마: `{ summary3: string[3], risk_level, risk_reason, action, amount, due_date }`
- 멀티모달=핵심(OCR 라이브러리 제거). 구조화 출력=핵심(prefill `{` 강제).

**Flutter 스택**
`image_picker`(카메라/갤러리) · `dio`(API, 타임아웃·재시도) · `dart:convert`(base64/jsonDecode) · `flutter_tts`(음성 낭독) · `provider`(로딩/결과/에러 3상태) · `flutter_image_compress`(업로드 속도) · `shared_preferences`(문서함). API 키는 `--dart-define` 주입(깃 커밋 금지), 슬라이드엔 "실서비스는 백엔드 프록시" 1줄.

**2일(무박2일) 빌드 계획**
- **Day0 저녁 (2~3h)** — 프로젝트 셋업, 패키지 추가, API 키 연결, curl로 멀티모달+JSON 호출 1건 성공시켜 **프롬프트 확정**(여기가 80%).
- **Day1 오전 (4~5h)** — 사진→base64→호출→JSON 파싱 end-to-end. 실제 고지서/스미싱 샘플로 신호등 정확도 튜닝.
- **Day1 오후 (4~5h)** — 화면3 신호등 결과 UI + 큰 글씨/고대비 디자인 토큰. 로딩 애니메이션.
- **Day1 밤 (3h)** — flutter_tts 음성, 화면4·5(왜 위험/할 일+전화), 문서함.
- **Day2 오전 (3~4h)** — 홈/빈·에러 상태, 폴백(미리 분석한 결과 캐시).
- **Day2 오후 (3~4h)** — 라이브 리허설(심사위원 즉석 촬영 + 안전망), 슬라이드/60초 영상/기획서.

**심사 기준 적합성**
- **사회적 가치(30):** 김순자의 "이거 사기야?" 전화 장면으로 공감. 정보격차→사기 취약성 프레임. 고독·불안 정조준.
- **창의성(25):** 번역·요약을 넘어 '진위 판단(신호등)'까지 — 어르신이 가장 무서워하는 지점을 정조준한 발상.
- **AI 활용(20):** 요약+쉬운말+사기추론이 단 1회 호출에. "이게 다 한 번의 Claude 호출" 흐름도 1장.
- **완성도(15):** 음성 안내·문서함·에러 처리까지 진짜 서비스. 어르신 맞춤 큰글씨 UI.
- **발표(10):** 신호등 빨강이 뜨는 0.5초가 모든 걸 말한다.

**7분 발표 데모 시나리오**
1. **(0:00~1:30) 문제** — "어르신께 도착하는 건 작은 글씨 고지서, 그 사이 섞인 스미싱." 실제 스미싱 문자 띄우고 "이게 진짜일까요 가짜일까요?"
2. **(1:30~2:30) 사람** — 김순자. 매번 자식에게 "이거 사기야?" 전화하는 의존과 미안함.
3. **(2:30~3:00) 서비스** — "찍기만 하면 신호등으로 답합니다."
4. **(3:00~5:30) 라이브 (클라이맥스)** — 심사위원께 "지금 폰의 문자나 고지서를 찍어주세요"(안전망: 준비한 스미싱 캡처). → 로딩 → **빨강 '사기 의심'** + 큰 글씨 요약 + 손주 톤 음성 "이건 누르지 마세요". 흐름도 1장.
5. **(5:30~7:00) 가치** — "찰칵 효도는 어르신이 다시는 무섭게 종이를 받지 않게 합니다." 확장(노인 1인가구·복지관 비치) 비전.

**60초 소개영상 앵글**
'두려움→안심'의 반전. 0~12s 우편함 앞 작은 글씨에 한숨 짓는 손. 12~25s 스미싱 문자에 손가락이 머뭇. 25~45s 찰칵 효도로 촬영→**빨강 사기 신호등**+음성 "누르지 마세요"→안도하는 표정. 45~60s 로고 + "찰칵 효도 — 찍으면 압니다. Powered by Claude." (캡컷 자막, Canva 로고)

**리스크 & 완화**
| 리스크 | 완화 |
|---|---|
| 사기 false negative(진짜 사기를 초록) → 역효과 | 프롬프트에 "확신 없으면 yellow" 규칙, 빨강/노랑 보수적 분류 |
| 라이브 호출 지연/실패 | 미리 분석한 스미싱·고지서 캐시 폴백, 로딩 애니메이션 |
| JSON 깨짐 | prefill `{` 강제 + 재시도 1회 + 요약만이라도 표시 |
| "AI가 법적/금융 판단" 오해 | "보조 도구, 확실치 않으면 가족·기관 확인" 톤 명시 |

**차별점 (vs Canva 목업 팀)**
다른 팀이 그림 목업을 넘길 때, 우리는 심사위원의 진짜 문자를 그 자리에서 찍어 **라이브로 빨강 사기 신호등**을 띄운다. 실제 인식 + 실제 추론이 동시에 일어나는 이 순간이 곧 완성도와 AI 활용의 증명이다.

## 3. 🌏 다문화 트랙 — 오뭐 (오늘 뭐래?)
> "사장님·관리자·관공서가 보낸 한국어 메시지, 캡처 한 번이면 '나에게 뭘·언제까지·얼마를' 모국어로 풀어주고 위험까지 짚어준다."

**한 줄 정리:** 이주노동자가 받은 한국어 카톡/문자/공지를 캡처하면 모국어 의역 + 핵심 추출 + 위험 플래그 + 한국어 답장까지 만들어주는 앱.

**문제 정의**
국내 체류 외국인 250만+(2024). 다수가 농어촌·제조 현장에서 매일 한국어 카톡·문자·공지를 받지만, 줄임말·사투리·행정용어·모호한 존댓말 때문에 '무슨 말인지'를 넘어 '나에게 뭘 시키는지, 언제까지인지, 돈으로 손해 볼 게 있는지'를 못 잡는다. 번역기는 직역만 해서 집합시간·마감·임금 차감 같은 '행간'을 통째로 놓친다. 그 결과 집합을 놓쳐 일당을 깎이거나, 부당한 정산에 영문도 모른 채 끄덕이고 만다.

**페르소나**
스레이몸(35세, 캄보디아, 한국 2년차, 농촌 비닐하우스 노동자). 회화는 조금 되지만 읽기·존댓말·행정용어가 약하다. 매일: 사장님 "내일 새벽 5시 집합, 비 오면 연락", 관리자 "이번 달 숙소비 정산", 행정 "신고기한 안내". 고통점: 집합 시간 헷갈려 놓침, 정산 금액 못 따짐, 매번 동료에게 캡처 보내 묻기 미안함, 정중한 답장 못 씀.

**HMW (How Might We)**
한국어를 못 읽는 이주노동자가, 받은 한국어 메시지를 캡처 한 번으로 '뭘·언제까지·얼마를'까지 모국어로 이해하고, 돈·기한 위험을 알아채며, 정중한 답장까지 바로 보낼 수 있게 하려면?

**가치 제안**
번역기가 멈추는 곳에서 오뭐는 시작한다. 캡처 1장이면 모국어 의역 + 핵심(할 일/일시/장소/금액) 카드 + 돈·계약·기한 위험 플래그 + 정중한 한국어 답장 초안까지 한 화면에. '이해'를 넘어 '행동'과 '자기 보호'를 만든다.

**MVP 기능** (2일 빌드 스코프)
| 기능 | 왜 필요 | Claude 사용 | Claude 역할 |
|---|---|---|---|
| 화면 캡처 입력 | 실제 행동(캡처해 묻기)을 그대로 | ✓ | 멀티모달로 이미지 속 한국어 읽기(OCR 흡수) |
| 모국어 의역(직역 아님) | '일당 깐다'→'늦으면 임금 차감' | ✓ | 줄임말·존댓말·화자관계 반영 의역 |
| 핵심 추출 카드 | 무엇/언제/어디/얼마 한눈에 | ✓ | 구조화 출력으로 action/datetime/amount 추출 |
| 위험 플래그(돈·계약·기한 ⚠️) | 약자 보호 = 사회가치 핵심 | ✓ | 임금체불·부당공제·과태료 신호 탐지 |
| 정중한 한국어 답장 1탭 생성 | 이해→행동, 오해 방지 | ✓ | 상황 맞춤 한국어 답장 2~3개 생성 |
| 히스토리 | 정산 분쟁 시 증거 | ✗ | — |

**화면 구성** (5장+)
1. **홈 / 캡처 입력** — 큰 버튼 "메시지 캡처 넣기", 최근 분석 미리보기.
2. **분석 로딩** — 캡처 + "AI가 읽고 있어요" 애니메이션.
3. **결과 통합 (핵심)** — 원문 / 모국어 의역 / 추출 카드 / 위험 배너 한 스크롤. 데모 클라이맥스.
4. **위험 상세** — 빨강 탭 시 "왜 위험한지" 모국어 설명 + 대응 팁.
5. **답장 생성** — 정중한 한국어 답장 후보 2~3개 + 모국어 해설 + 복사 버튼.
6. **히스토리** — 날짜·화자·요약·위험 뱃지.
7. **언어/설정** — 모국어 선택(크메르/베트남/네팔어), 데모 시 즉석 전환.

**Claude 연동 (구체)**
캡처 이미지 → base64 → image 블록 + 시스템 프롬프트로 **단 1회 멀티모달 호출** → JSON 반환 → 카드 렌더. 모델 `claude-opus-4-8`(정확도) / `claude-sonnet-4-6`(속도).

- 시스템 프롬프트 스케치:
  > "당신은 한국에서 일하는 이주노동자를 돕는 보조자입니다. 이미지는 노동자가 받은 한국어 카톡/문자/공지 캡처입니다. JSON으로만 답하세요: `ocr_text`(원문), `translation`(목표어 의역, 직역 금지), `summary_cards`(action_item/datetime/location/amount/deadline), `risks`([{level: red|yellow, type: 임금|계약|기한|기타, message_목표어}]), `reply_drafts`([{korean, note_목표어}] 2~3개). 화자 관계(사장/관리자/관공서)를 추정해 톤을 맞추세요. 본문에 없는 사실은 만들지 마세요."
- 한 호출에 **번역+구조화추출+위험탐지+작문 4능력**이 동시에 → AI 활용도 직격.

**Flutter 스택**
`image_picker` · `dio`/`http` · `dart:convert`(base64) · `provider`/`riverpod` · `hive`/`shared_preferences`(히스토리) · `flutter/services` Clipboard + `share_plus` · 비라틴 글리프용 Noto Sans 번들. 데모 타깃 모국어 1개(크메르) 고정, `target_lang`만 파라미터로 전환(앱 전체 i18n은 과스코프). 키는 `--dart-define`.

**2일(무박2일) 빌드 계획**
- **Day0 저녁 (2~3h)** — 셋업, curl로 멀티모달+JSON 1건 성공 → 프롬프트 확정(80%).
- **Day1 오전 (4~5h)** — 이미지→base64→호출→파싱 end-to-end. 의역/JSON/위험 민감도 튜닝.
- **Day1 오후 (4~5h)** — 화면3 통합 결과 UI(원문/의역/카드/위험 배너), 위험 색상.
- **Day1 밤 (3~4h)** — 화면5 답장+클립보드, 화면6 히스토리, 화면4 위험 상세.
- **Day2 오전 (3~4h)** — 홈/설정(target_lang), 크메르 폰트, 빈·에러·파싱실패 폴백.
- **Day2 오후 (3~4h)** — 라이브 리허설 + 캐시 폴백, 슬라이드/영상/기획서, 타이밍.

**심사 기준 적합성**
- **사회적 가치(30):** 250만+ 통계 + '동료에게 매번 캡처 보내 묻기' 장면. 임금체불·부당정산 = 약자가 가장 잘 당하는 지점 보호.
- **창의성(25):** '캡처해 묻기' 행동을 그대로 인터페이스화(학습비용 0) + 번역→이해→행동→자기보호 확장.
- **AI 활용(20):** 한 호출·한 화면에 4능력 가시화. "단순 번역기가 아니다"가 눈에 보임.
- **완성도(15):** 7화면 + 실서버 API + 빈/에러/히스토리/다국어 전환.
- **발표(10):** 심사위원 폰의 진짜 카톡 즉석 캡처 → 모국어 + 임금 위험 빨강 + 답장이 라이브로.

**7분 발표 데모 시나리오**
1. **(0~1:00) 문제** — "외국인 250만. 매일 받는 건 한국어 카톡." 실제 카톡 '늦으면 일당 깐다' 띄우고 "크메르어만 안다면?"
2. **(1:00~2:00) 사람** — 스레이몸. 동료에게 캡처 보내 묻고, 없으면 집합 놓쳐 일당 깎임.
3. **(2:00~2:30) 서비스** — "번역기는 직역만. 오뭐는 이해와 행동까지."
4. **(2:30~5:00) 라이브 (클라이맥스)** — 심사위원 폰 카톡 즉석 캡처 → 로딩 → 크메르어 의역 + '5시 집합' 카드 + 빨강 '⚠️ 임금 차감 주의' + 정중한 답장 1탭 복사. 흐름도 1장.
5. **(5:00~7:00) 가치** — "오뭐는 번역기가 아니라 약자의 방패." 확장(베트남·네팔어) 비전.

**60초 소개영상 앵글**
'캡처 한 장이 바꾸는 하루', 스레이몸 1인칭. 0~12s 비닐하우스 아침, 한국어 카톡 클로즈업. 12~22s 동료에게 보내려다 멈칫("매번 묻기 미안해요"). 22~40s 오뭐 실행→캡처→결과(크메르어 + 카드 + 빨강 위험 + 답장)→표정이 풀림. 40~52s '복사' 1탭→정시 출근, 일당 그대로. 52~60s 로고 "오뭐 — 번역을 넘어, 행동까지."

**리스크 & 완화**
| 리스크 | 완화 |
|---|---|
| 카톡 스크린샷 작은 글씨/말풍선 인식·행간 위험 탐지 흔들림 | 검증 샘플 카톡 3~4장 준비해 프롬프트 고정, 즉석은 보너스 |
| JSON 깨짐 | 출력 형식 강제 + 재시도 1회 + 번역 텍스트만이라도 표시 |
| 위험 오탐(정상을 위험으로) → 신뢰 손상 | '주의' 톤("~일 수 있으니 확인하세요"), 법률자문 아님 명시 |
| 스코프 폭증(다국어 전체) | 모국어 1개 고정, 단일 플로우에 올인 |

**차별점 (vs Canva 목업 팀)**
심사위원 폰 속 진짜 한국어 카톡을 즉석 캡처해 라이브 Claude 호출 한 번으로 모국어 의역 + 추출 카드 + 빨강 위험 + 복사 가능한 한국어 답장이 한 화면에 동시에 뜬다. 목업이 아닌 '지금 작동하는 AI'가 결정적 우위.

## 4. ♿ 배리어프리 트랙 — 쉬운고지서 (Easy Letter)
> "어려운 우편물 사진 한 장이면, 무슨 일을 언제까지 해야 하는지 알려드려요."

**한 줄 정리:** 발달장애인·저문해층이 행정/금융 우편물을 찍으면 한 문장 요약 + 쉬운말 토글 + 할 일 체크리스트 + 캘린더 등록까지 주는 앱.

**문제 정의**
행정·금융 문서의 관료적 문장과 작은 글씨는 발달장애인·저문해층에게 사실상 '읽을 수 없는 벽'이다. 국내 발달장애인 약 26만 명, 성인 최저 문해 수준 인구 200만+. 하지만 고지서·과태료·재계약 안내문에서 '누가/무엇을/얼마/언제까지/안 하면 어떻게 되는지'를 뽑아주는 도구는 없다. '글자는 보이는데 뜻을 모르는' 인지적 소외 탓에 기한을 놓쳐 연체·불이익을 반복하고, 매번 보호자에게 사진을 찍어 설명을 구걸해야 하는 무력감을 겪는다.

**페르소나**
김민수(28세, 경증 발달장애). 부모 집을 떠나 원룸 자립생활 시작. 마트 보조로 일하지만 흰 봉투(건강보험·통신·과태료·재계약)가 오면 가슴이 철렁. '본인부담금'·'납부기한'·'소멸시효' 같은 단어로 가득해 무슨 뜻인지 모른다. 서랍에 미루다 연체 통지를 받고서야 어머니에게 사진을 보낸다. "또 물어봐서 죄송한데…"가 입에 붙었다.

**HMW (How Might We)**
어려운 행정·금융 우편물을 사진 한 장으로 찍었을 때, 김민수가 누구에게도 묻지 않고 '무엇을·얼마를·언제까지'를 스스로 이해하고 기한 안에 행동하게 하려면?

**가치 제안**
관료적 문장의 우편물을 찍으면 5초 만에 "8월 15일까지 건강보험료 6만 8천 원을 내야 해요" 한 문장 카드로. 원문↔쉬운말 토글, 할 일 1·2·3, 안 하면 어떻게 되는지, 전화·캘린더 버튼까지 한 화면에. 의존을 자립으로 바꾸는 '읽기 동반자'.

**MVP 기능** (2일 빌드 스코프)
| 기능 | 왜 필요 | Claude 사용 | Claude 역할 |
|---|---|---|---|
| 우편물 사진 1장 | 진짜 행동(찍어 묻기)을 입력으로, 인지부담 0 | ✗ | — |
| Claude 비전 단일 호출 구조화 추출 | OCR+의미+요약+행동+쉬운말 1회에 | ✓ | summary/easyText/originalText/amount/dueDate/actions/consequence JSON |
| 한 문장 핵심 요약 카드 | 가장 먼저 알아야 할 단 하나의 답 | ✓ | summary 한 문장 압축 |
| 원문↔쉬운말 토글 | AI 변환을 눈앞에서 증명 = AI활용 핵심 | ✓ | originalText·easyText 함께 반환 |
| 할 일 1·2·3 + '안 내면?' | 이해→행동 | ✓ | actions[] + consequence |
| 전화하기 / 캘린더 기한 추가 | 마지막 한 걸음까지 자립적 완료 | ✗ | — |

**화면 구성** (5장+)
1. **홈 / 우편물 찍기** — 큰 카메라 버튼 + "우편물을 찍어보세요", 지난 기록.
2. **분석 중** — 사진 + "AI가 편지를 읽고 있어요" 5초 애니메이션.
3. **핵심 요약 카드 (메인)** — "8월 15일까지 6만 8천 원" 큰 글씨 + 금액·기한 뱃지.
4. **원문↔쉬운말 토글 (핵심 데모)** — 어려운 원문과 큰 글씨·이모지 쉬운말을 즉석 비교.
5. **내가 할 일 + 안 하면?** — 체크리스트 1·2·3 + "안 내면 연체료가 붙어요".
6. **액션/완료** — '전화하기', '캘린더에 8월 15일 추가' → 실제 전화·캘린더 앱 열림.
7. **내 우편함** — 처리한 우편물 + 다가오는 기한 정렬(자립의 증거).

**Claude 연동 (구체)**
사진 → base64 → image 블록 + 텍스트 지시로 **단 1회 호출** → JSON → 카드 1:1 매핑. 모델 `claude-opus-4-8`/`claude-sonnet-4-6`, max_tokens ~1024, temp 0.2.

- 시스템 프롬프트 스케치:
  > "당신은 글 읽기가 어려운 한국 성인을 돕는 '쉬운고지서' 도우미입니다. 첨부 행정·금융 우편물 사진을 읽고 JSON으로만 답하세요. `easyText`는 초등 2학년도 이해하게 한 문장에 한 내용, 어려운 한자어·법률용어 금지, 이모지 1개, 존댓말. 금액·날짜는 본문에서 정확히 인용, 불확실하면 null(추측 금지). dueDate는 YYYY-MM-DD. consequence는 겁주지 말고 사실만 부드럽게. JSON 외 금지."
- JSON 스키마: `{ summary, documentType, amount, dueDate, originalText, easyText, actions[], consequence }`. prefill `{` 강제.

**Flutter 스택**
`image_picker` · `http`/`dio` · `dart:convert` · `provider` · `url_launcher`(tel:) · `add_2_calendar`(네이티브 캘린더 삽입, 가장 빠름) · `google_fonts`(Noto Sans KR, 큰 가독성) · `flutter_image_compress`(1280px 다운스케일). `LetterResult` 모델 1개로 8필드 매핑. 데모용 고지서 1장은 오프라인 캐시 응답으로도 준비.

**2일(무박2일) 빌드 계획**
- **Day1 0~2h** — 셋업, 패키지, 'Hello 호출' 텍스트 응답 성공.
- **Day1 2~5h** — 멀티모달 호출 완성, 실제 종이 고지서로 OCR 검증.
- **Day1 5~8h** — 시스템 프롬프트+JSON 튜닝, 고지서 3~4종(건강보험·과태료·통신·재계약)으로 쉬운말 품질 반복(점수 핵심).
- **Day1 8~12h** — 화면3·4(요약 카드 + 원문↔쉬운말 토글), 디자인 토큰(큰 폰트·고대비).
- **Day2 12~15h** — 화면1·2(홈/로딩), 1→2→3→4 연결.
- **Day2 15~18h** — 화면5·6(체크리스트, 전화·캘린더 실제 동작), 화면7 가볍게.
- **Day2 18~21h** — 이미지 리사이즈·에러·캐시 폴백, 라이브 리허설 3회.
- **Day2 21~24h** — 60초 영상·슬라이드·대본·비상 녹화.
> 시간 부족 시 컷: 화면7, google_fonts. 절대 사수: 사진→Claude→요약카드→쉬운말 토글→캘린더 추가.

**심사 기준 적합성**
- **사회적 가치(30):** 발달장애 26만·저문해 200만 통계 + "또 물어봐서 죄송한데…" 한마디. '의존을 자립으로'.
- **창의성(25):** '쉬운말 토글 = AI 변환 before/after를 눈으로'. 행정문서를 답+체크리스트+결과+버튼으로 재구성.
- **AI 활용(20):** OCR+의미+쉬운말+행동을 1회 호출 JSON으로. '사진1→Claude1→JSON8필드→화면6요소' 흐름도.
- **완성도(15):** 실제 구동 + 캘린더 등록까지. 큰 폰트·고대비·이모지 당사자 맞춤.
- **발표(10):** 즉석 종이 고지서 촬영이 클라이맥스. 비상 녹화로 리스크 차단.

**7분 발표 데모 시나리오**
1. **(0~1:30) 문제** — 흰 봉투 클로즈업 "이 종이를 못 읽는다면?" 26만·200만 통계 → '인지적 소외'.
2. **(1:30~2:30) 사람** — 김민수, 자립 시작했지만 우편물마다 어머니에게 "또 죄송한데…".
3. **(2:30~3:00) 서비스** — "사진 한 장이면 됩니다."
4. **(3:00~5:30) 라이브 (클라이맥스)** — 무대의 실제 종이 건강보험 고지서 즉석 촬영 → 5초 → "8월 15일까지 6만 8천 원" 카드 → 원문↔쉬운말 토글 → 할 일 1·2·3 + "안 내면 연체료" → '캘린더에 추가' 눌러 실제 일정 꽂힘. 흐름도 1장.
5. **(5:30~7:00) 가치** — "민수가 다시는 죄송하다 말하지 않게. 의존을 자립으로." 확장(전기·수도, 노인·이주민).

**60초 소개영상 앵글**
'봉투의 공포→혼자 해냄'. 0~10s 어두운 톤, 깨알 글씨 "글자는 보이는데 뜻을 모릅니다". 10~20s 봉투를 미루고 '엄마 이거 뭐야?'. 20~35s 앱 촬영→"AI가 읽고 있어요"→"8월 15일까지 6만 8천 원" 카드, 토글 탭. 35~50s 체크리스트+캘린더 등록, 살짝 미소 "이제 혼자 할 수 있어요". 50~60s "쉬운고지서 — 사진 한 장이면 끝. Powered by Claude 비전."

**리스크 & 완화**
| 리스크 | 완화 |
|---|---|
| 금액·마감일 오인식/추측 → 행정문서라 치명 | "불확실하면 null, 추측 금지" 강제 + 원문 토글로 대조, 데모용 고지서 사전 검증 |
| 라이브 네트워크/지연 | 동일 고지서 오프라인 캐시 + 비상 녹화 영상 임베드 |
| JSON 파싱 실패 | prefill `{` + try/catch + 재시도, '다시 찍어주세요' 폴백 |
| 당사자 UI 부적합 지적 | 큰 폰트·고대비·한 화면 한 정보·이모지·존댓말 원칙, 페르소나에서 도출했음을 발표에서 연결 |

**차별점 (vs Canva 목업 팀)**
무대에서 건넨 진짜 종이 고지서를 그 자리에서 촬영하면 관료적 문장이 5초 만에 'AI가 진짜로 읽고' 한 문장으로 바뀌고, 원문↔쉬운말 토글과 캘린더 자동 등록까지 실제로 동작한다. 목업은 '될 거예요', 우리는 '지금 됩니다'.

## 5. 🧑‍⚕️ 돌봄 트랙 — 교대인계 (Gyodae-Ingye)
> "말로 쏟아낸 하루가, 다음 근무자를 위한 인계장으로 — 뒤죽박죽 음성 한 덩어리를 입소자별 인계 카드 여러 장으로."

**한 줄 정리:** 요양보호사가 근무 끝에 두서없이 말하면, Claude가 입소자별·카테고리별 인계 카드로 분해하고 위험·할 일을 추론해주는 앱.

**문제 정의**
돌봄 현장의 진짜 기록 부담은 '단건 기록'이 아니라 '교대 인계'다. 머릿속엔 12명의 정보가 시간·사건 순으로 뒤섞여 있는데, 이를 구조화된 인계장으로 옮기는 작업이 가장 무겁고, 누락되면 낙상·투약 사고로 직결된다. 게다가 손글씨/타이핑에 약한 50~60대 베테랑은 '1건씩 입력' 디지털 솔루션에서 구조적으로 소외된다(입력이 더 일). 현장이 원하는 건 '한 번에 다 쏟아내고 AI가 정리'다.

**페르소나**
요양원 야간 요양보호사 김순자(54). 오전 7시 퇴근 직전, 입소자 12명의 밤사이 변동을 손글씨 인계장에 옮기느라 30분째 묶여 있다. 글쓰기가 느리고 한자어 의료용어 표기가 어려워 늘 누락이 생긴다. 디지털 앱은 '입력이 더 일'이라 외면했다. 고통점: 기록에 잡아먹히는 퇴근, 누락이 곧 사고, 디지털 소외감.

**HMW (How Might We)**
손글씨·타이핑에 약한 베테랑 요양보호사가, 근무 끝에 두서없이 말하기만 하면 다음 근무자가 바로 쓸 수 있는 입소자별 인계장과 위험 알림이 자동으로 만들어지게 하려면?

**가치 제안**
교대인계는 '입력'을 '말하기'로 바꾼다. 2~3분 의식 흐름대로 말하면, 한 덩어리 음성이 입소자별·카테고리별 인계 카드 여러 장 + '낮번 우선 확인 TOP3' 위험 리스트로 정돈된다. 받아쓰기가 아니라 추론이다.

**MVP 기능** (2일 빌드 스코프)
| 기능 | 왜 필요 | Claude 사용 | Claude 역할 |
|---|---|---|---|
| 음성 녹음(2~3분) + 화면 텍스트 표시 | 베테랑이 쓰는 대신 말하게 | ✗(STT는 디바이스) | — (speech_to_text가 받아쓰기) |
| 텍스트 붙여넣기 폴백 | STT 실패 대비 데모 안정성 | ✗ | — |
| 입소자별 자동 분리 | 섞인 한 덩어리를 N명으로 | ✓ | 다중 엔티티 분해(이름/별칭 추적) |
| 카테고리 구조화(활력·식사·배변·수면·통증·투약·낙상) | 다음 근무자가 바로 읽게 | ✓ | 항목별 구조화 JSON |
| 누락 위험 추론 → 액션 승격 | '혈압 높았다는데 약 확인 언급 없음' | ✓ | 명시 안 된 위험 추론, action으로 승격 |
| 낙상·투약 위험 플래그 + TOP3 | 환자 안전 = 사회가치 핵심 | ✓ | 위험 키워드 감지, risk_level + 우선순위 |

**화면 구성** (5장+)
1. **홈 / 녹음 시작** — 큰 마이크 버튼 "오늘 있었던 일 말하기" + 텍스트 입력 폴백 진입.
2. **녹음 중 / 인식 텍스트** — 실시간 STT 텍스트 표시, 탭 보정 가능.
3. **정리 중** — "AI가 입소자별로 나누고 있어요" 애니메이션(분해 연출).
4. **인계 보드 (핵심)** — 입소자별 카드 그리드, 카테고리 칩, 낙상·투약 빨강 점등.
5. **카드 상세** — 한 입소자의 시간순 변동 + 근거 문장 + '내일 확인' 액션.
6. **낮번 우선 확인 TOP3** — 가장 위험한 액션 3개 상단 고정.
7. **인계장 내보내기** — 텍스트/이미지로 다음 근무자에게 공유(공유 버튼).

**Claude 연동 (구체)**
디바이스 STT(`speech_to_text`)가 음성→텍스트. 그 **비정형 장문 텍스트**를 시스템 프롬프트와 함께 Anthropic Messages API에 **단 1회 호출** → 입소자별·카테고리별 구조화 JSON → 카드 렌더. 모델 `claude-opus-4-8`(추론 품질) / `claude-sonnet-4-6`(속도).

- 시스템 프롬프트 스케치:
  > "당신은 요양 인계를 돕는 보조자입니다. 입력은 요양보호사가 교대 끝에 두서없이 말한 받아쓰기 텍스트입니다. 여러 입소자 이야기가 시간·사건 순으로 뒤섞여 있습니다. JSON으로만 답하세요: `residents`([{name, items:[{category: 활력|식사|배변|수면|통증|투약|기타, note, time, source_quote}], risks:[{level: red|yellow, type: 낙상|투약|기타, reason}], actions:[{text, priority}]}]), `top3_actions`(가장 위험·우선 액션 3개). **음성에 없는 사실은 절대 만들지 말 것**(환각 금지). 단, 명시되지 않았지만 후속 확인이 필요한 위험(예: '혈압 높았다는데 투약 언급 없음')은 action으로 승격하되 추정임을 표시."
- JSON 스키마: 위 `residents[] + top3_actions`. prefill `{` 강제.
- **AI 흐름 명시:** STT(디바이스) → Claude API 구조화·위험추론·액션생성 **1회 호출**. 단순 받아쓰기와 다른 '추론'임을 슬라이드에 강조.

**Flutter 스택**
`speech_to_text`(디바이스 STT, 무료) · `dio`/`http`(Claude API) · `dart:convert` · `provider`/`riverpod` · `hive`/`shared_preferences`(인계 기록) · `share_plus`(내보내기). **텍스트 붙여넣기 폴백 경로 필수**(STT 무대 실패 대비). 키는 `--dart-define`.

**2일(무박2일) 빌드 계획**
- **Day0 저녁 (2~3h)** — 셋업, speech_to_text 권한·동작 확인, curl로 '뒤섞인 텍스트→입소자별 JSON' 1건 성공 → 프롬프트 확정.
- **Day1 오전 (4~5h)** — STT 텍스트 → Claude 호출 → JSON 파싱 end-to-end. 시드 시나리오(입소자 3~4명) 음성으로 분해 품질 튜닝.
- **Day1 오후 (4~5h)** — 화면4 인계 보드(카드 그리드 + 카테고리 칩 + 위험 점등), 화면5 상세.
- **Day1 밤 (3~4h)** — 화면6 TOP3, 화면2 녹음/인식 텍스트 + 탭 보정, 텍스트 폴백.
- **Day2 오전 (3~4h)** — 홈/내보내기/기록, 빈·에러·STT실패 폴백.
- **Day2 오후 (3~4h)** — 사전 검증 시나리오 음성으로 라이브 리허설, 슬라이드/영상/기획서.

**심사 기준 적합성**
- **사회적 가치(30):** 인계 누락→낙상·투약 사고라는 '환자 안전' 서사(가장 무거움) + 디지털 소외된 50대 종사자를 음성으로 포용 → 'AI Re-Local' 메타 주제 정통.
- **창의성(25):** '단건 기록'이 아닌 '교대 인계'를 조준한 문제 재정의 + '한 덩어리→N장 분해 + 누락 위험 추론'.
- **AI 활용(20):** 받아쓰기가 아니라 '비구조 장문→다중 엔티티 구조화+위험 추론'. Claude 추론력을 가장 또렷이 증명.
- **완성도(15):** 실제 구동 + 위험 점등·TOP3·내보내기까지.
- **발표(10):** 음성 한 덩어리가 카드로 쪼개져 정렬·점등되는 연출이 "받아쓰기 아니다"를 0.5초에 각인.

**7분 발표 데모 시나리오**
1. **(0~1:30) 문제** — 손글씨 인계장에 묶인 퇴근. "누락 한 줄이 낙상·투약 사고로." 50대 종사자의 디지털 소외.
2. **(1:30~2:30) 사람** — 김순자, 30분째 12명을 옮겨적으며 누락 걱정.
3. **(2:30~3:00) 서비스** — "말만 하세요. 정리는 AI가."
4. **(3:00~5:30) 라이브 (클라이맥스)** — 사전 검증 시나리오 음성(또는 텍스트 폴백) 입력 → "AI가 나누고 있어요" → 화면에서 조각이 입소자별 카드로 분해·정렬, 낙상·투약 빨강 점등, 하단 'TOP3' 자동 생성. "이건 받아쓰기가 아니라 추론입니다." 흐름도 1장.
5. **(5:30~7:00) 가치** — "교대인계는 베테랑의 손이 아니라 입을 쓰게 합니다. 사고를 막고, 소외를 포용으로." 확장(보육 알림장, 장애인 활동지원).

**60초 소개영상 앵글**
'손이 묶인 퇴근→입으로 끝내는 인계'. 0~12s 새벽, 펜으로 인계장 빼곡히 적다 한숨, 시계 7:00. 12~22s 자막 "12명을 다 적기엔 시간이 없어요". 22~42s 앱 켜고 말하기→화면에서 카드가 입소자별로 분해·정렬, 낙상 빨강 점등, TOP3 생성. 42~52s 다음 근무자가 카드 보며 "오늘은 김씨 어르신 낙상 주의군요". 52~60s "교대인계 — 말하면 인계장이 됩니다. Powered by Claude."

**리스크 & 완화**
| 리스크 | 완화 |
|---|---|
| 디바이스 STT 오인식(소음·사투리·의료용어) | 사전 검증 시나리오 음성 사용 + 인식 텍스트 탭 보정 + **텍스트 붙여넣기 폴백** |
| 위험 플래그 환각(없는 위험/누락) → 의료 책임 | "음성에 없는 사실 만들지 말 것" 제약, 추정 표시, "AI 보조·최종 확인은 사람" 명시 |
| 라이브 호출 지연/실패 | 사전 분석 결과 캐시 + 비상 녹화 |
| JSON 깨짐 | prefill `{` + 재시도 + graceful 폴백 |

**차별점 (vs Canva 목업 팀)**
다른 팀이 그림 목업을 낼 때, 우리는 무대에서 두서없는 음성 한 덩어리를 실제로 입력해 라이브 Claude 호출로 입소자별 카드 분해 + 위험 점등 + TOP3가 눈앞에서 생성되는 걸 보여준다. 입력이 음성이라 4개 트랙 중 AI 추론력이 가장 차별적으로 드러난다.

## 6. CEO 최종 추천 — 딱 하나 고른다면

무박 2일·Flutter 1인 기준, **우선순위 추천:**

1. **🥇 쉬운고지서 (배리어프리) — 최고 추천.** 내부 점수 1위(106). 입력이 **사진 1장**이라 무대에서 깨질 단일 실패점이 거의 없고(STT·실시간 없음), '원문↔쉬운말 토글'이 AI 변환을 심사위원 눈에 그대로 박는다. 캘린더 자동 등록이라는 물리적 마무리가 완성도를 증명. **가장 안전하면서 점수 천장이 높다.**

2. **🥈 오뭐 (다문화).** 점수 2위(103). 쉬운고지서와 같은 사진/캡처 엔진인데 **번역+추출+위험+답장 4능력**이 한 화면에 떠 AI 활용도(20)를 가장 강하게 친다. '임금 위험 플래그'의 사회적 메시지가 묵직. 쉬운고지서와 엔진을 공유하므로 **둘 중 무엇을 골라도 코드 자산 재사용 가능.**

3. **🥉 교대인계 (돌봄).** 차별성 1등(음성→추론). "받아쓰기가 아니라 추론"이라는 한 방이 강력하지만, STT가 유일한 리스크 → **텍스트 폴백 필수**. 안정성 한 단계 아래.

4. **찰칵 효도 (실버).** 가장 따뜻하고 데모가 극적(빨강 사기 신호등). 단, 사기 오판(false negative)이 신뢰를 깰 수 있어 보수적 분류 규칙이 생명.

### 결정적 한 수 — 멀티 트랙 전략
실버·다문화·배리어프리는 **동일 엔진**(`사진/캡처 → Claude 멀티모달 1회 → JSON → 카드`)이다. 따라서:
- **Day0~Day1 오전:** 공용 엔진(`image_picker → base64 → Anthropic 호출 → JSON 파싱 → 카드 위젯`)을 1번만 단단히 만든다.
- **Day1 오후~Day2:** 시스템 프롬프트 + 카드 UI만 트랙별로 바꿔 끼운다.
- 결과: **메인 1개를 완성도 높게**, 시간 남으면 두 번째 트랙을 같은 엔진으로 즉시 파생.

> **나의 추천: 쉬운고지서(배리어프리)를 메인으로 빌드 → 같은 엔진으로 오뭐(다문화)를 보너스로.** 안전성·점수·재사용성·발표 임팩트의 균형이 최적.

## 7. 공통 빌드 키트 (어느 트랙이든)

**공용 Claude 호출 골격**
- 엔드포인트: `POST https://api.anthropic.com/v1/messages`
- 헤더: `x-api-key`(--dart-define 주입), `anthropic-version: 2023-06-01`, `content-type: application/json`
