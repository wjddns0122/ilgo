## 1. TL;DR

**읽고**는 어려운 한국어 글(종이 우편물 또는 휴대폰 화면 메시지)을 사진 한 장으로 찍으면, 사용자가 읽을 수 있는 말(**쉬운 한국어** 또는 **모국어**)로 풀어주고, **위험(사기·임금·기한)을 신호등으로 짚어주고**, **할 일/답장까지** 정리해주는 앱이다. 하나의 AI 이해 엔진이 발달장애·저문해 청년(김민수)과 영어권 이주노동자(마리아)를 **같은 흐름**으로 포용한다. **배리어프리·다문화 두 트랙에 동시 출전**하며, 두 페르소나를 동등하게 내세워 "같은 앱, 다른 사람"을 증명한다. 데모 모국어는 **영어**(심사위원이 출력을 직접 검증 가능).

---

## 2. 문제 정의 (Why now)

한국어 글을 받지만 그 **뜻·위험·할 일**을 읽지 못하는 사람들이 있다.

- 발달장애인 약 **26만 명**, 성인 최저 문해 수준 인구 **200만 명+** → 행정·금융 우편물의 관료적 문장이 '읽을 수 없는 벽'.
- 체류 외국인 **250만 명+**(2024) → 줄임말·존댓말·행정용어가 섞인 한국어 메시지를 못 읽어 집합을 놓치거나 부당 정산에 동의.

기존 도구(번역기·요약기)는 '글자'만 바꾸고 **'행간'(기한·임금 위험·다음 행동)을 통째로 놓친다.** 그래서 매번 가족·동료에게 "이거 뭐야?"를 물어야 하는 **의존과 위축**이 남는다.

> 핵심: 김민수와 스레이몸은 다른 사람 같지만 **같은 벽** 앞에 있다 — (1) 무슨 뜻인지, (2) 나에게 위험한지, (3) 뭘 해야 하는지를 모른다.

## 3. 목표 & 비목표

### Goals
- G1. 어려운 한국어 글을 **사진 한 장**으로 이해 가능하게(쉬운 한국어/모국어).
- G2. 단순 번역을 넘어 **위험 경고 + 행동 안내**까지 제공.
- G3. **하나의 AI 엔진**으로 두 사용자(배리어프리/다문화)를 포용 → "AI for Everyone" 구현.
- G4. **실제 구동 Flutter 앱 + 라이브 AI 호출**로 데모(목업 아님).

### Non-Goals (이번 MVP에서 안 함)
- N1. 계정/로그인·서버·DB·결제·푸시.
- N2. 다국어 전체 i18n(데모는 **영어 1개**; 엔진은 다국어 가능).
- N3. 법률/세무 자문 수준의 정확도 보장(보조 도구로 포지셔닝).
- N4. 백엔드 프록시·키 은닉(데모는 클라이언트 직접 호출, 실서비스 과제로 발표).

## 4. 성공 지표 (Success Metrics)

해커톤 맥락이므로 **심사 배점 + 데모 완수**를 성공 지표로 삼는다.

| 지표 | 목표 |
|---|---|
| 심사 — 사회적 가치·공감 (30) | 두 페르소나 서사로 상위권 |
| 심사 — 창의성 (25) | "한 엔진=플랫폼" 발상 + 원문↔풀이 토글 |
| 심사 — AI 활용 (20) | 단 1회 호출에 5능력 가시화 + 흐름도 |
| 심사 — 완성도 (15) | 2개 모드 실제 동작 + 캘린더/답장 |
| 심사 — 발표 (10) | 7분에 두 시나리오(민수·마리아) + 심사위원 즉석 입력 |
| 데모 — 응답시간 | 촬영→결과 ≤ 7초 |
| 데모 — 무중단 | 라이브 실패 시 폴백으로 0 크래시 |

## 5. 대상 사용자 & 페르소나

> **두 트랙 동시 출전** — 두 페르소나는 동등한 주인공이다.

**P-A · 김민수 (28, 경증 발달장애, 자립 청년)** — *배리어프리 트랙*
원룸 자립 시작. 흰 봉투(건강보험·과태료·통신·재계약)가 오면 '본인부담금'·'소멸시효'에 막혀 서랍에 미루다 연체. "또 물어봐서 죄송한데…"가 입에 붙음. → **모드: 쉬운 한국어.**

**P-B · 마리아 (32, 필리핀, 제조 공장 노동자)** — *다문화 트랙*
영어는 읽지만 한국어 읽기는 어렵다. 사장님 "내일 새벽 5시 집합, 늦으면 일당 깐다", 관리자 "숙소비 정산" 같은 한국어 메시지를 못 읽어 집합을 놓치거나 부당 정산에 끄덕이고, 정중한 한국어 답장을 못 쓴다. → **모드: 모국어(영어) + 한국어 답장.**

## 7. 제품 원칙

1. **사진 한 장이 입력의 전부.** 학습 비용 0.
2. **한 화면, 한 핵심.** 가장 먼저 답(한 문장)부터.
3. **이해를 넘어 행동까지.** 풀이 + 위험 + 다음 행동.
4. **정직한 AI.** 불확실하면 단정하지 않음, 원문으로 대조 가능.
5. **한 엔진, 여러 사용자.** 모드는 출력 언어만 다름.

## 11. 데이터 모델 (ReadResult)

```jsonc
{
  "doc_type": "고지서",                 // 고지서|공지|메시지|계약|기타
  "summary_one_line": "8월 15일까지 건강보험료 6만 8천 원을 내야 해요",
  "original_text": "…원문(한국어)…",
  "explained_text": "…쉬운 한국어 또는 모국어 풀이…",
  "cards": { "what": "건강보험료 납부", "when": null, "where": null,
             "amount": "68,000원", "deadline": "2026-08-15" },
  "risks": [{ "level": "yellow", "type": "기한",
              "message": "8월 15일이 지나면 연체료가 붙을 수 있어요" }],
  "actions": [{ "text": "은행 앱이나 지로로 6만 8천 원을 내세요", "priority": 1 }],
  "consequence": "기한이 지나면 연체료가 더해질 수 있어요",
  "reply_drafts": [{ "korean": "네, 기한 안에 납부하겠습니다.", "note_in_lang": "…모국어 해설…" }]
}
```
**규칙:** 금액·날짜 정확 인용, 불확실 시 null(추측 금지). 위험 확신 없으면 yellow. 없는 사실 생성 금지.

**UserProfile(로컬):** `{ "output_mode": "easy_korean", "lang": "ko" }` — 모국어 모드 예: `{ "output_mode": "native", "lang": "en" }`

## 12. AI / 엔진 요구사항

- **호출:** Anthropic Messages API `POST /v1/messages` — 이미지 블록(base64) + 시스템 프롬프트, **1회**.
- **모델:** `claude-opus-4-8`(정확도) / `claude-sonnet-4-6`(속도). `temperature 0.2`, `max_tokens ~1200`, assistant prefill `{`로 JSON 강제.
- **시스템 프롬프트(요지):** "한국어 글을 못 읽는 사람을 돕는 '읽고' 도우미. `output_mode`가 easy_korean이면 초등 2학년 수준 쉬운 한국어, native면 `lang`으로 직역 아닌 의역. 지정 키(doc_type, summary_one_line, original_text, explained_text, cards, risks, actions, consequence, reply_drafts(native만))의 JSON으로만 답. 위험은 스미싱·임금차감·부당공제·과태료·신고기한; 확신 없으면 yellow. 금액·날짜는 정확 인용, 불확실 시 null, 없는 사실 금지."
- **AI 가시화(20점):** 1회 호출에 OCR(멀티모달)+의역/쉬운말+추출+위험추론+(native)작문 = 5능력. 발표에 `이미지1→호출1→JSON→화면5요소` 흐름도.

| 성능 | 촬영→결과 ≤ 7초, 타임아웃 30초 |
| 신뢰성 | 재시도 1회 + 사전 캐시 폴백으로 데모 무중단 |
| 보안 | API 키 `--dart-define`(깃 커밋 금지), 실서비스는 프록시 명시 |
| 개인정보 | 기본 로컬 처리, 데모는 클라이언트 직접 호출·외부 저장 없음 |
| 접근성 | 큰 글씨·고대비·TTS·터치 48dp+ |
| 권한 | 카메라, 캘린더 쓰기 (마이크·위치 미사용) |
