## 1. 핵심 결정

**목데이터 응답을 만들지 않는다.**

백엔드는 실제 `POST /v1/analyses` 분석에서 사기/스미싱 문자를 인식하고, 기존 `Analysis` 응답 스키마로 위험과 할 일을 반환해야 한다.

추가 백엔드 책임은 2개다.

1. 스미싱/사기 의심 메시지 분석 품질을 보장한다.
2. 위험 타입별 도움처를 `/v1/help-contacts`에서 반환한다.

DB 대형 변경은 하지 않는다. 기존 `risks.type`, `risks.level`, `risks.message`, `risks.detail`, `risks.steps`, `actions[]`를 활용한다.

---

## 2. API 범위

| Method | Path | 변경 |
|---|---|---|
| POST | `/v1/analyses` | 스미싱/사기 판단 규칙 강화 |
| GET | `/v1/help-contacts` | `risk_type` 쿼리 추가 |

변경하지 않는 것:

- `/v1/analyses/{id}`
- `/v1/analyses`
- `/v1/analyses/{id}/actions/{actionId}`
- `/v1/analyses/{id}/replies:regenerate`
- 인증/로그인 흐름

## 3. `POST /v1/analyses` 요구사항

### BE-AN-01. 스미싱/사기 위험 반환

백엔드는 이미지 속 문자가 아래 조건을 포함하면 사기/스미싱 위험으로 판단해야 한다.

#### 위험 신호

| 신호 | 예시 |
|---|---|
| 기관 사칭 | 국민건강보험, 국세청, 경찰, 법원, 택배사, 은행 등 |
| 낯선 URL | 공식 도메인이 아닌 링크, 단축 URL, 오타 도메인 |
| 압박 문구 | 오늘까지, 즉시 납부, 압류, 고발, 계정 정지 |
| 금전 요구 | 미납금, 수수료, 과태료, 보증금 |
| 개인정보 요구 | 주민번호, 계좌, 카드, 인증번호, 비밀번호 |
| 앱 설치 유도 | 보안 앱 설치, 확인 앱 설치 |

### BE-AN-02. 응답 스키마

기존 `Analysis` 스키마를 유지한다.

스미싱 분석 성공 시 필수/권장 값:

| 필드 | 요구 |
|---|---|
| `doc_type` | `메시지` |
| `summary_one_line` | "사기/스미싱일 수 있다"가 한눈에 드러나야 함 |
| `original_text` | OCR 원문 |
| `explained_text` | 쉬운 한국어 또는 사용자 언어 풀이 |
| `consequence` | 링크를 누를 경우 생길 수 있는 피해 |
| `cards.what` | 어떤 문자/안내인지 |
| `cards.amount` | 본문 금액이 있으면 정확히 추출 |
| `cards.deadline` | 실제 날짜가 있으면 `YYYY-MM-DD`, "오늘"만 있으면 null 권장 |
| `risks[].level` | 명확한 사칭/URL/압박이 있으면 `red`, 애매하면 `yellow` |
| `risks[].type` | 표준은 `사기`, 더 구체적으로는 `스미싱` 허용 |
| `risks[].message` | 링크 미클릭 + 공식 채널 확인 안내 |
| `risks[].detail` | 왜 위험한지 2~4문장 설명 |
| `risks[].steps` | 사용자가 할 행동 2~4개 |
| `actions[]` | 링크 누르지 않기, 공식 채널 확인, 118 상담 |
| `reply_drafts` | easy_korean 모드에서는 빈 배열 |

### BE-AN-04. 프롬프트 요구사항

분석 시스템 프롬프트에 아래 규칙을 추가한다.

```text
스미싱/피싱/사기 의심 문자는 별도 위험으로 판단한다.
공식 기관명, 금융기관명, 택배사, 공공기관을 사칭하면서 낯선 URL,
단축 URL, 오타 도메인, 앱 설치 유도, 개인정보/금전 입력 유도,
"오늘까지", "즉시", "압류", "계정 정지" 같은 압박 문구가 있으면
red 또는 yellow 위험으로 표시한다.

확실하지 않으면 "사기입니다"라고 단정하지 말고
"사기/스미싱이 의심돼요. 공식 채널로 확인하세요"라고 말한다.

행동에는 다음 중 관련 있는 것을 포함한다.
- 문자 안의 링크를 누르지 않기
- 공식 앱/대표번호/공식 홈페이지로 직접 확인하기
- 118에 상담하기
- 이미 눌렀다면 비밀번호/카드/계좌 정보를 입력하지 않기

금액과 날짜는 본문에 있는 그대로만 추출한다.
본문에 없는 사실은 만들지 않는다.
```

### BE-AN-05. 위험 타입 표준

백엔드는 아래 값을 우선 사용한다.

| 상황 | `risks.type` |
|---|---|
| 스미싱/피싱/사칭 링크 | `사기` |
| 더 구체적으로 표현하고 싶을 때 | `스미싱` |
| 임금 삭감/체불 | `임금` |
| 근로계약/숙소비 공제 | `계약` |
| 납부/마감 문제 | `기한` |
| 과태료/벌금 | `과태료` |
| 불명확 | `기타` |

프론트가 기존 위험 타입을 포함 검색으로 처리하므로 `스미싱`도 허용한다. 다만 발표 안정성을 위해 사기성 문자는 가급적 `사기`로 통일한다.

## 4. `GET /v1/help-contacts` 요구사항

### BE-HC-01. `risk_type` 쿼리 추가

현재:

```http
GET /v1/help-contacts?lang=ko&doc_type=메시지
```

변경:

```http
GET /v1/help-contacts?lang=ko&doc_type=메시지&risk_type=사기
GET /v1/help-contacts?lang=en&risk_type=임금
```

### 요청 파라미터

| 이름 | 필수 | 설명 |
|---|---|---|
| `lang` | no | 사용자 언어. `ko`, `en` 등 |
| `doc_type` | no | `메시지`, `고지서`, `계약` 등 |
| `risk_type` | no | `사기`, `스미싱`, `임금`, `계약`, `과태료`, `기한`, `기타` |

### 응답 스키마

기존 `HelpContactsResponse`를 유지한다.

```json
{
  "items": [
    {
      "id": "kisa_118",
      "title": "스미싱·해킹 상담",
      "org": "KISA/보호나라",
      "phone": "118",
      "category": "사기",
      "emergency": false
    }
  ]
}
```

### BE-HC-02. 정렬 규칙

`risk_type`이 있을 때는 해당 위험에 가장 적합한 연락처를 첫 번째로 반환한다.

| risk_type | 1순위 | 2순위 |
|---|---|---|
| `사기`, `스미싱`, `피싱`, `보이스피싱` | `kisa_118` | `police_112` |
| `임금`, `급여`, `일당`, `부당공제` | `moel_1350` | `immigration_1345` |
| `계약`, `숙소비`, `공제` | `moel_1350` | `immigration_1345` |
| `건강보험` | `nhis_1577_1000` | `kisa_118` |
| `과태료`, `기한` | 일반 도움처 | 긴급 아님 |
| 기타/없음 | 기본 목록 | 없음 |

### BE-HC-03. 기본 연락처 데이터

MVP는 DB 테이블 없이 static config로 구현해도 된다.

| id | title | org | phone | category | emergency |
|---|---|---|---|---|---|
| `kisa_118` | 스미싱·해킹 상담 | KISA/보호나라 | 118 | 사기 | false |
| `moel_1350` | 임금·노동 상담 | 고용노동부 | 1350 | 임금 | false |
| `immigration_1345` | 외국인종합안내센터 | 법무부 | 1345 | 외국인 | false |
| `nhis_1577_1000` | 건강보험 문의 | 국민건강보험공단 | 1577-1000 | 건강보험 | false |
| `police_112` | 긴급 · 범죄 신고 | 경찰 | 112 | 긴급 | true |

### BE-HC-04. fallback 정책

| 상황 | 응답 |
|---|---|
| `risk_type` 없음 | 기본 도움처 목록 |
| 알 수 없는 `risk_type` | 기본 도움처 목록 |
| `lang` 미지원 | 한국어 기본 목록 |
| 연락처 phone 없음 | 해당 항목은 반환하지 않거나 phone null 허용. 프론트가 버튼 숨김 |

## 5. DB/마이그레이션

### 5.1 기본 방침

DB 변경은 필수가 아니다.

필수 조건:

- `risks.type`이 `사기` 또는 `스미싱` 값을 저장할 수 있어야 한다.
- `risks.detail`과 `risks.steps`가 있다면 그대로 저장/반환한다.
- `help-contacts`는 static config로 처리해도 된다.

### 5.2 enum/CHECK 제약이 있는 경우

만약 DB나 코드 enum이 위험 타입을 제한한다면 아래 값을 허용한다.

```text
사기
스미싱
임금
계약
부당공제
기한
과태료
건강보험
기타
```

가능하면 DB enum보다 varchar + 서버 검증을 권장한다. 해커톤 MVP에서는 새 위험 타입 때문에 배포가 막히면 안 된다.

## 6. 보안/개인정보 요구사항

| 항목 | 요구 |
|---|---|
| URL | 서버는 URL을 직접 방문하거나 실행하지 않는다. 텍스트 패턴으로만 판단한다. |
| 악성 링크 | 응답에 원문 URL이 포함될 수 있으나, 프론트가 자동으로 열지 않는다. |
| 로그 | 원문 전체를 INFO 로그에 남기지 않는다. 디버그 필요 시 마스킹한다. |
| 개인정보 | 주민번호/계좌/인증번호는 저장/로그에서 마스킹 권장. |
| 판단 톤 | "사기 확정" 대신 "사기/스미싱 의심" 표현 우선. |
| 상담 안내 | 118/1350은 상담/확인 안내로만 사용. 자동 신고 아님. |

## 7. 에러 처리

### `POST /v1/analyses`

| 상황 | HTTP | code | message |
|---|---|---|---|
| 이미지 없음 | 400 | `INVALID_IMAGE` | 이미지를 읽을 수 없습니다. |
| 이미지 너무 큼 | 413 | `PAYLOAD_TOO_LARGE` | 이미지가 너무 큽니다. |
| AI 응답 파싱 실패 | 422 | `AI_PARSE_FAILED` | 분석에 실패했어요. 다시 시도해 주세요. |
| AI 호출 실패 | 502 | `AI_PROVIDER_ERROR` | 잠시 후 다시 시도해 주세요. |
| Rate limit | 429 | `RATE_LIMITED` | 요청이 많아요. 잠시 후 다시 시도해 주세요. |

### `GET /v1/help-contacts`

이 API는 가능하면 실패하지 않는다.

| 상황 | 응답 |
|---|---|
| 알 수 없는 risk_type | 200 + 기본 목록 |
| 빈 데이터 | 200 + fallback 기본 목록 |
| 내부 오류 | 500. 단, static config라면 발생 가능성을 낮춘다. |

## 8. 테스트 요구사항

### Unit

- `risk_type=사기` → 첫 번째 연락처 `kisa_118`.
- `risk_type=스미싱` → 첫 번째 연락처 `kisa_118`.
- `risk_type=임금` → 첫 번째 연락처 `moel_1350`.
- `risk_type=계약` → `moel_1350`, `immigration_1345` 포함.
- 알 수 없는 `risk_type` → 기본 목록 반환.
- 연락처 목록에 phone 없는 항목이 있어도 직렬화 실패 없음.

### Integration

- `GET /v1/help-contacts?risk_type=사기` 200.
- `GET /v1/help-contacts?risk_type=임금` 200.
- `POST /v1/analyses`에 스미싱 이미지 입력 시 `risks[].type`이 `사기` 또는 `스미싱`.
- 같은 응답에 `actions[]`가 링크 미클릭/공식 확인/118 상담 중 최소 2개를 포함.

### Prompt/Eval Fixture

아래 케이스를 고정 fixture로 둔다.

1. 공공기관 사칭 + 낯선 URL + 압박 문구 → red.
2. 택배 사칭 + 앱 설치 유도 → red/yellow.
3. 실제 공공요금 고지서 + 공식 도메인 없음 → 사기로 단정하지 않음.
4. 임금 삭감 문자 → 임금 위험, 1350 도움처.

## 9. 수용 기준

백엔드 작업은 아래가 모두 충족되면 완료다.

- [ ] `POST /v1/analyses`가 스미싱성 문자 이미지를 `red` 또는 최소 `yellow` 위험으로 반환한다.
- [ ] 스미싱 응답에 `type = 사기` 또는 `스미싱`이 들어간다.
- [ ] 스미싱 응답의 actions에 링크 미클릭과 공식 채널 확인이 들어간다.
- [ ] 스미싱 응답의 risk detail/steps가 프론트 위험 상세에서 바로 쓸 수 있다.
- [ ] `GET /v1/help-contacts?risk_type=사기`가 118을 첫 번째로 반환한다.
- [ ] `GET /v1/help-contacts?risk_type=임금`이 1350을 첫 번째로 반환한다.

## 10. 프론트와 맞출 계약

프론트가 의존하는 핵심 계약은 아래다.

```text
risk.type contains 사기/스미싱 → 118
risk.type contains 임금/계약/부당공제 → 1350
help-contacts first item = 가장 추천하는 CTA
analysis.risks[].steps = 위험 상세의 행동 가이드
analysis.actions[] = 결과 화면 할 일 체크
```

이 계약만 지켜지면 프론트는 목데이터 없이 실서버 결과로 데모를 구성할 수 있다.
