# 읽어드림 (Ilgeo-Drim) — 최종 통합 기획서

> **쉬운고지서 + 오뭐 통합 디벨롭** · AI Re-Local 해커톤 · Flutter 1인 · 무박 2일
> **한 문장 피치:** "어려운 한국어 글(종이든 화면이든), 사진 한 장이면 당신이 읽을 수 있는 말로 풀어주고, 위험까지 짚어주고, 할 일까지 정리해드려요."

---

## 0. 왜 이 둘을 합치는가 — 통합의 핵심 통찰

쉬운고지서와 오뭐는 **겉보기엔 다른 앱, 속은 같은 엔진**이었다.

| | 쉬운고지서 (배리어프리) | 오뭐 (다문화) |
|---|---|---|
| 사용자 | 발달장애·저문해 김민수 | 이주노동자 스레이몸 |
| 입력 | 행정 우편물 **사진** | 한국어 카톡 **캡처** |
| 출력 | **쉬운 한국어** + 할 일 + 캘린더 | **모국어** + 위험 + 답장 |
| 엔진 | 멀티모달 1회 호출 → JSON → 카드 | 멀티모달 1회 호출 → JSON → 카드 |

**통찰:** 김민수와 스레이몸은 전혀 다른 사람 같지만 **똑같은 벽** 앞에 서 있다.
한국어 글이 도착했는데 — (1) **무슨 뜻인지** 모르고, (2) **나에게 위험한지** 모르고, (3) **뭘 해야 하는지** 모른다.
기존 도구(번역기·요약기)는 '글자'만 바꾸고 **'행간'(위험·기한·행동)을 통째로 놓친다.**

> **그래서 통합 제품 "읽어드림"은 단품 2개가 아니라, '글을 못 읽는 모두'를 포용하는 하나의 이해 엔진이다.**
> 입력이 종이든 화면이든, 사용자가 발달장애인이든 이주민이든 — 같은 엔진이 사용자에 맞춰 답한다. 이것이 **"AI for Everyone"을 문자 그대로** 구현한다.

### 합쳐도 초점이 흐려지지 않는 이유 (CEO 점검)
- **단일 플로우는 동일하다:** `캡처 → 이해 → 행동`. 모드는 **출력 언어만** 다르다.
- **사용자는 자기 모드 하나만 본다.** 기능이 늘어난 게 아니라, 같은 기능이 두 사람을 포용한다.
- **발표는 한 페르소나(김민수)로 시작**해 공감을 확보하고, 마지막 45초에 "같은 앱이 이 사람도 돕습니다(스레이몸)"로 **플랫폼임을 증명**한다. 공감 → 확장성 순서라 초점이 살아있다.

## 1. 출전 트랙 결정 (정하면 바꿀 수 있음)

해커톤은 트랙 하나를 골라 출전한다. **추천: ♿ 배리어프리 트랙으로 출전.**

**왜 배리어프리:**
- 내부 채점 1위(106) + 입력이 **사진 한 장**이라 무대에서 깨질 단일 실패점이 거의 없음(STT·실시간 없음).
- "쉬운고지서"는 또렷한 앵커 페르소나(김민수, 자립청년)가 있어 **사회적 가치·공감(30점)** 서사가 강하다.
- 마지막에 **모국어 모드(스레이몸)** 를 라이브로 보여 "이건 단품이 아니라 플랫폼"임을 증명 → **창의성(25)·완성도(15)** 가산.

**🔁 다문화 트랙으로 출전한다면 (flip note):** 리드 페르소나를 스레이몸으로 바꾸고, 데모 메인을 카톡 캡처→모국어+임금 위험+답장으로, 마지막 45초에 김민수 고지서를 "확장 증명"으로 보여주면 된다. **엔진·코드는 100% 동일** — 발표 동선과 첫 페르소나만 스왑.

## 2. 제품 정의

**이름:** 읽어드림 (Ilgeo-Drim) — "읽어 드릴게요" + dream
- 대체 후보: **풀어줌**(풀이+풀어주다), **이지(Easy)**, **다읽어**

**태그라인:** "어려운 글, 한 줄로 읽어드려요."

**가치 제안**
종이 고지서든 휴대폰 속 한국어 메시지든, 사진 한 장이면 **읽어드림**이 ① 핵심을 한 문장으로 요약하고 ② 당신이 읽을 수 있는 말(쉬운 한국어 또는 모국어)로 풀어주고 ③ 사기·임금·기한 같은 위험을 신호등으로 짚어주고 ④ 할 일 체크리스트 또는 정중한 한국어 답장까지 만들어준다. **'이해'를 넘어 '행동'과 '자기 보호'를 준다.**

**HMW (통합)**
한국어 글을 받지만 그 뜻·위험·할 일을 읽지 못하는 사람(발달장애·저문해·이주민)이, 사진 한 장으로 자기가 읽을 수 있는 말로 이해하고, 위험을 알아채고, 다음 행동까지 스스로 하게 하려면?

## 3. 두 페르소나, 한 엔진

**페르소나 A — 김민수 (28, 경증 발달장애, 자립 청년)** · *배리어프리, 리드*
원룸 자립 시작. 흰 봉투(건강보험·과태료·재계약)가 오면 '본인부담금'·'소멸시효'에 막혀 서랍에 미루다 연체. "또 물어봐서 죄송한데…"가 입에 붙음. → **모드: 쉬운 한국어.**

**페르소나 B — 스레이몸 (35, 캄보디아, 비닐하우스 노동자)** · *다문화, 확장 증명*
사장님 "내일 새벽 5시 집합, 늦으면 일당 깐다", 관리자 "숙소비 정산". 줄임말·존댓말 못 읽어 집합 놓치고 일당 깎임, 부당 정산에 끄덕임, 정중한 답장 못 씀. → **모드: 모국어(크메르어) + 답장.**

> 둘은 **같은 홈 화면, 같은 '찍기' 버튼, 같은 Claude 호출**을 쓴다. 다른 건 온보딩에서 고른 **출력 모드**뿐이다.

## 4. 통합 기능 설계

### 4.1 사용자 모드 (온보딩 1회 선택)
- **모드 ①: 쉬운 한국어** — 큰 글씨·짧은 문장·이모지. (저문해·발달장애·어르신)
- **모드 ②: 내 모국어** — 언어 선택(크메르/베트남/네팔/영어…). 모국어 풀이 + 한국어 답장 생성. (이주민)
- 모드는 **출력 언어/톤만** 결정. 입력·분석·화면 구조는 동일.

### 4.2 공통 입력
- **카메라**(종이 우편물) 또는 **갤러리/캡처**(화면 메시지) — 단일 파이프라인.

### 4.3 공통 출력 (한 화면)
| 요소 | 쉬운 한국어 모드 | 모국어 모드 |
|---|---|---|
| ① 핵심 한 문장 | "8월 15일까지 6만 8천 원 내야 해요" | (모국어로 동일) |
| ② 원문 ↔ 풀이 토글 | 원문 ↔ **쉬운 한국어** | 원문 ↔ **모국어 의역** |
| ③ 핵심 추출 카드 | 무엇/얼마/언제까지 | 무엇/언제/어디/얼마 |
| ④ 위험 신호등 🟢🟡🔴 | 사기·과태료·기한 | **임금·계약·기한** |
| ⑤ 행동 | 할 일 1·2·3 + **캘린더 추가** | + **정중한 한국어 답장 1탭 복사** |
| 음성(TTS) | 핵심 한 줄 낭독 | (선택) |

### 4.4 MVP 스코프 규율 (2일)
- **모드 ① (쉬운 한국어): 풀(full)** — 요약·토글·할 일·캘린더·음성. (배리어프리 우승작 그대로)
- **모드 ② (모국어): 라이트(lite)** — 모국어 풀이 + 위험 + 답장 1탭. 플랫폼 증명에 충분.
- **만들지 않는 것:** 7개 언어 전체 i18n(데모는 크메르 1개), 계정/서버 인증, 양쪽 풀 히스토리.

## 5. 화면 구성 (8장)

1. **온보딩 / 모드 선택** — "쉬운 한국어로 볼까요? / 내 언어로 볼까요?" (언어 선택)
2. **홈 / 입력** — 큰 버튼 2개: "📷 종이 찍기" / "🖼️ 화면 캡처 넣기". 아래 최근 기록.
3. **분석 중** — 입력 이미지 + "AI가 읽고 있어요" 애니메이션 (라이브 호출 가시화).
4. **결과 메인 (핵심)** — 핵심 한 문장 큰 글씨 + 위험 신호등 배너. 데모 클라이맥스.
5. **원문 ↔ 풀이 토글** — 어려운 원문과 쉬운말/모국어를 즉석 비교. **AI 변환을 눈으로 증명.**
6. **핵심 카드 + 할 일 / 안 하면?** — 추출 카드 + 체크리스트 + 결과 설명 + **캘린더 추가** 버튼.
7. **[모국어 모드] 한국어 답장 생성** — 정중한 답장 후보 2~3개 + 모국어 해설 + **복사** 버튼.
8. **내 보관함** — 처리한 글 목록, 신호등 색·다가오는 기한 정렬 (자립의 증거).

## 6. Claude 연동 — 하나의 호출, 두 사용자

핵심 차별점: **입력·사용자·문서유형이 무엇이든 단 1회 Claude 멀티모달 호출**이 모든 걸 처리한다. 모드는 system 프롬프트의 파라미터로만 분기한다.

**흐름:** 이미지(종이/캡처) → base64 → `image` 블록 + system 프롬프트(+ user_profile) → **1회 호출** → JSON 슈퍼셋 → UI가 모드에 맞는 필드만 렌더 → (쉬운 모드) flutter_tts 낭독.

- **모델:** `claude-opus-4-8`(정확도 우선) / 데모 속도용 `claude-sonnet-4-6`. `temperature: 0.2`, assistant prefill `{`로 JSON 강제.

**시스템 프롬프트 스케치 (모드 분기):**
> "당신은 한국어 글을 못 읽는 사람을 돕는 '읽어드림' 도우미입니다. 첨부 이미지는 사용자가 받은 한국 행정·금융 우편물 또는 메시지(카톡/문자/공지) 화면입니다.
> `user_profile`: `{output_mode: "easy_korean" | "native", lang: "<ISO>"}`.
> 다음을 수행해 **JSON으로만** 답하세요:
> - `doc_type`: 고지서 | 공지 | 메시지 | 계약 | 기타
> - `summary_one_line`: 핵심 한 문장 (output_mode가 easy_korean이면 초등 2학년 수준 쉬운 한국어, native면 lang으로)
> - `original_text`: 이미지 속 한국어 원문
> - `explained_text`: easy_korean이면 쉬운 한국어 풀이(짧은 문장·이모지·존댓말), native면 lang으로 **직역 아닌 의역**(줄임말·존댓말 뉘앙스 반영)
> - `cards`: {what, when, where, amount, deadline} (없으면 null)
> - `risks`: [{level: green|yellow|red, type: 사기|임금|계약|기한|과태료|기타, message}] — 스미싱·임금차감·부당공제·신고기한 등 사용자가 손해 볼 신호. **확신 없으면 red 단정 금지, yellow로.**
> - `actions`: [{text, priority}] 할 일
> - `consequence`: 안 하면 생기는 일 (겁주지 말고 사실만)
> - `reply_drafts`: native 모드일 때만, 정중한 한국어 답장 2~3개 [{korean, note_in_lang}]
> **금액·날짜는 본문에서 정확히 인용, 불확실하면 null(추측 금지). 본문에 없는 사실은 만들지 마세요.**"

**통합 JSON 스키마:**
```json
{
  "doc_type": "고지서",
  "summary_one_line": "...",
  "original_text": "...",
  "explained_text": "...",
  "cards": { "what": null, "when": null, "where": null, "amount": null, "deadline": null },
  "risks": [{ "level": "red", "type": "임금", "message": "..." }],
  "actions": [{ "text": "...", "priority": 1 }],
  "consequence": "...",
  "reply_drafts": [{ "korean": "...", "note_in_lang": "..." }]
}
```

**AI 활용도(20점) 직격:** 한 번의 호출에 **OCR(멀티모달 흡수) + 의역/쉬운말 변환 + 구조화 추출 + 위험 추론 + (모국어 모드)작문** 5가지 능력이 한 화면에 보인다. 발표에 `이미지1 → Claude 1회 → JSON → 화면 5요소` 흐름도 1장.

## 7. Flutter 스택

| 용도 | 패키지 |
|---|---|
| 입력 | `image_picker` (카메라/갤러리) |
| 이미지 처리 | `dart:convert`(base64) · `flutter_image_compress`(1280px 다운스케일) |
| API 호출 | `dio` (타임아웃·재시도) — Anthropic Messages API 직접 POST |
| 상태관리 | `provider` (로딩/결과/에러 3상태) |
| 음성 | `flutter_tts` (쉬운 모드 낭독) |
| 액션 | `url_launcher`(tel:) · `add_2_calendar`(캘린더) · Clipboard/`share_plus`(답장 복사) |
| 저장 | `shared_preferences` 또는 `hive` (보관함) |
| 폰트 | `google_fonts` Noto Sans KR + 모국어 글리프(Noto Sans Khmer) |

- API 키는 `--dart-define` 주입(깃 커밋 금지). 슬라이드에 "실서비스는 백엔드 프록시" 1줄.
- 핵심 모델 클래스 `ReadResult` 하나로 JSON 전 필드 매핑.

## 8. 2일(무박2일) 빌드 계획

전제: 숙련 Flutter 1인. **공용 엔진을 1번만 단단히 만들고, 모드는 출력만 분기.**

**Day 0 저녁 (2~3h) — 셋업 + 프롬프트 확정 (80%가 여기)**
- 프로젝트 생성, 패키지 추가, API 키 연결.
- curl/Postman으로 **멀티모달 + JSON 출력** 호출 1건 성공. 종이 고지서 1장 + 카톡 캡처 1장으로 양쪽 검증. 통합 프롬프트 확정.

**Day 1 오전 (4~5h) — 공용 엔진**
- `image_picker → base64 → dio POST → JSON 파싱(ReadResult)` end-to-end 1줄 성공.
- 실제 고지서 3~4종 + 카톡 샘플로 쉬운말/의역 품질·위험 탐지 튜닝.

**Day 1 오후 (4~5h) — 결과 화면(공통)**
- 화면4 결과 메인(한 문장 + 신호등), 화면5 원문↔풀이 토글, 화면6 카드+할 일+캘린더.
- 디자인 토큰: 큰 폰트·고대비·둥근 카드.

**Day 1 밤 (3~4h) — 모드 ② lite + 음성**
- 온보딩 모드 토글(화면1), `output_mode/lang`을 호출 파라미터로.
- 화면7 한국어 답장 생성+복사(모국어 모드), flutter_tts 낭독(쉬운 모드).

**Day 2 오전 (3~4h) — 마감·폴백**
- 화면2 홈(입력 2버튼), 화면3 로딩, 화면8 보관함(가볍게).
- 빈/에러/네트워크 실패/JSON 파싱 실패 폴백 + **사전 분석 결과 캐시**.

## 9. 심사 기준 적합성 (100점)

- **사회적 가치·공감 (30):** 두 페르소나로 "전혀 다른 사람, 같은 벽"을 보여줌. 발달장애 26만·저문해 200만 + 이주민 250만. "또 물어봐서 죄송한데…" / "매번 동료에게 캡처 보내 묻기". 정보격차가 사기·임금 피해로 이어지는 구조적 약점 보호.
- **아이디어 창의성 (25):** 단일 기능이 아니라 **'글 못 읽는 모두'를 포용하는 이해 엔진(플랫폼)**. '원문↔풀이 토글 = AI 변환을 눈으로'. 번역→이해→행동→자기보호로 가치사슬 확장.
- **AI 활용도 (20):** **단 1회 Claude 멀티모달 호출**에 OCR+의역/쉬운말+추출+위험추론+작문 5능력. 모드는 파라미터 분기. 흐름도 1장 + 라이브 호출 장면.
- **결과물 완성도 (15):** 실제 구동 Flutter 앱, **2개 모드가 진짜 동작** + 캘린더 등록·답장 복사까지. 빈/에러/보관함 등 '서비스다움'.
- **발표 전달력 (10):** 7분 1개 시나리오 + "같은 앱, 다른 사람" 반전 + 심사위원 즉석 입력. 비상 녹화로 리스크 차단.

> **통합이 단품보다 강한 지점:** 사회가치·AI활용은 동급으로 유지하면서, **창의성(플랫폼 발상)** 과 **완성도(2모드 실동작)** 에서 한 단계 위로 올라간다.

## 10. 7분 발표 데모 시나리오

1. **(0:00~1:30) 문제 — "같은 벽"**
   김민수와 스레이몸 사진을 나란히. "전혀 다른 사람입니다. 그런데 같은 벽 앞에 있습니다." 흰 봉투 / 한국어 카톡을 띄우고 "이 글을 못 읽는다면?" 통계(26만·200만·250만).
2. **(1:30~2:30) 사람 — 김민수 (리드)**
   자립 시작, 우편물마다 어머니에게 "또 죄송한데…". 의존과 위축.
3. **(2:30~3:00) 서비스 한 줄**
   "읽어드림. 사진 한 장이면, 당신이 읽을 수 있는 말로 풀어드려요."
4. **(3:00~5:00) 라이브 ① — 쉬운 한국어 (클라이맥스 1)**
   무대의 **실제 종이 건강보험 고지서 즉석 촬영** → 5초 → "8월 15일까지 6만 8천 원" 카드 → **원문↔쉬운말 토글** → 할 일 1·2·3 + "안 내면 연체료" → **'캘린더에 추가'** 눌러 실제 일정이 꽂힘.
5. **(5:00~5:45) 라이브 ② — "같은 앱, 다른 사람" (플랫폼 증명)**
   모드를 **모국어(크메르)** 로 전환 → 심사위원 폰의 **한국어 카톡 캡처** → 크메르어 의역 + **🔴 임금 차감 주의** + 정중한 한국어 답장 1탭 복사.
6. **(5:45~6:15) AI 과정**
   "방금 두 장면이 **전부 같은 Claude 멀티모달 1회 호출**입니다." 흐름도 1장.
7. **(6:15~7:00) 가치**
   "읽어드림은 김민수가 다시는 죄송하다 말하지 않게, 스레이몸이 부당한 정산에 끄덕이지 않게 합니다. **AI for Everyone — 글을 못 읽는 모두를 위해.**" 확장(어르신 사기 = 같은 엔진).

> **질의 대비:** 정확도(불확실 시 null·추측 금지, 원문 토글로 대조), 개인정보(데모는 클라이언트 호출, 실서비스 백엔드 프록시), 위험 오탐('주의' 톤, 보조 도구임 명시).

## 11. 60초 소개영상 앵글

'같은 벽, 같은 해방'. 두 사람의 교차 편집.
- **0~10s** 김민수: 흰 봉투, 깨알 글씨 클로즈업. 자막 "글자는 보이는데, 뜻을 몰라요."
- **10~20s** 스레이몸: 비닐하우스, 한국어 카톡. 자막 "매번 누군가에게 물어야 했어요."
- **20~40s** 둘 다 **읽어드림** 실행 → 촬영/캡처 → "AI가 읽고 있어요" → (김민수)"8월 15일까지 6만 8천 원" 쉬운말 카드 / (스레이몸) 크메르어 + 🔴 임금 주의 + 답장. 두 얼굴이 풀린다.
- **40~52s** 김민수 캘린더 등록 / 스레이몸 답장 복사·전송. 자막 "이제 혼자 할 수 있어요."
- **52~60s** 로고 + "**읽어드림 — 어려운 글, 한 줄로 읽어드려요. Powered by Claude.**" (캡컷 자막·전환, Canva 로고)

## 13. 차별점 (vs 경쟁팀)

다른 팀은 **하나의 페르소나용 단일 기능을 Canva 클릭목업**으로 낸다.
우리는 **실제 구동 Flutter 앱**에서, 심사위원이 건넨 진짜 종이 고지서를 그 자리서 찍어 쉬운말로 바꾸고 캘린더에 일정을 꽂은 뒤, **같은 앱을 모국어 모드로 돌려** 한국어 카톡을 크메르어 + 임금 위험 + 답장으로 풀어낸다.

> 목업은 "이렇게 될 거예요"를 말한다. 읽어드림은 **"지금, 두 사람 모두에게 됩니다"** 를 보여준다.
> 이 한 엔진이 글 못 읽는 모두를 포용한다는 증명이 곧 완성도(15)·AI활용(20)·창의성(25)의 격차다.
