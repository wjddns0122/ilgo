# 읽고 (Ilgo) — 팀 업무 분담 (개발 외 파트)

> 개발(앱)은 제가 맡습니다. 이 문서는 **개발 외 팀원분들**이 바로 시작할 수 있게 정리한 할 일 목록이에요.
> 각 역할 카드에 **목표 · 할 일 체크리스트 · 산출물 · 저(개발)에게 넘겨줄 것 · 왜 점수가 되는지**가 있습니다.
> 역할 카드는 인원수에 맞게 **합치거나 나눠도** 됩니다.

---

## 0. 30초 브리핑 (전원 먼저 읽기)

- **만드는 것:** **읽고** — 어려운 한국어 글(종이 고지서 / 휴대폰 문자·카톡)을 사진 한 장 찍으면, **쉬운 한국어 또는 영어로 풀어주고, 위험(사기·임금·기한)을 신호등으로 짚어주고, 할 일·답장까지** 정리해주는 앱.
- **누구를 위해 (두 명, 동등):**
  - **김민수 (28, 발달장애·저문해)** → 배리어프리 트랙. 행정 우편물을 못 읽어 매번 가족에게 물어봄.
  - **마리아 (32, 필리핀 노동자, 영어 사용)** → 다문화 트랙. 한국어 문자를 못 읽어 집합을 놓치거나 부당 정산에 동의.
- **우리만의 무기:** 다른 팀은 Canva 그림 목업. 우리는 **진짜 작동하는 앱 + 라이브 AI**. → "지금 됩니다"를 보여줌.
- **심사 배점(100):** 사회가치·공감 **30** / 창의성 **25** / AI 활용 **20** / 완성도 **15** / 발표 **10**.
- **자세한 내용:** `읽고_기능명세서.md`(쉬운 버전), `읽고_PRD.md`(상세). 급하면 이 브리핑만 봐도 OK.

> **모든 자료는 "김민수·마리아 두 사람을 돕는다"는 이야기로 수렴시켜 주세요.** 그게 점수의 핵심(공감 30점)입니다.

## 2. 역할별 상세

### A. 자료조사 & 사회적 근거 📊 (→ 사회가치 30)

**목표:** 발표에서 "이 문제가 진짜 크고 심각하다"를 **숫자와 출처**로 증명.

**할 일 (체크리스트)**
- [ ] 발달장애인 등록 인구 최신 수치 (출처: 보건복지부/장애인 통계)
- [ ] 성인 문해력 최저 수준 인구 (출처: 국가문해능력조사/OECD PIAAC)
- [ ] 국내 체류 외국인 수, 특히 노동자 비중 (출처: 법무부 출입국·외국인정책 통계)
- [ ] 이주노동자 임금체불·부당 정산 관련 통계/사례 (고용노동부·기사)
- [ ] 어르신·취약계층 스미싱/보이스피싱 피해 통계 (경찰청·금감원) — 위험 신호등 근거
- [ ] "행정 문서가 어렵다"는 근거 (공공언어·쉬운 우리말 관련 자료)
- [ ] **기존 서비스 비교:** 정부24, 번역앱(파파고/구글), 배리어프리 앱 등 — **왜 이들이 '행간(위험·기한·행동)'을 못 푸는지** 1~2줄씩

**산출물:** 근거 정리표 (항목 / 수치 / 출처 / 링크 / 발표에 쓸 한 줄). Google Docs 또는 .md.
**저(개발)에게 넘길 것:** 위험 신호등에 쓸 "사기·임금 위험 패턴" 예시(실제 문구) 몇 개.
**주의:** 숫자엔 **반드시 출처**를 달 것(심사위원 질문 대비). 애매하면 "약 ~만 명" + 출처.

### B. 사용자 조사 & 현장 관찰 🧑‍🤝‍🧑 (→ 공감 30)

**목표:** 두 페르소나(김민수·마리아)를 **진짜 사람처럼** 만들고, 현장 관찰로 공감 서사 확보.

**할 일**
- [ ] 김민수 페르소나 구체화: 하루 일과, 우편물 받을 때 감정, 실제 겪는 불편 (필요 시 복지관/자립지원 사례 참고)
- [ ] 마리아 페르소나 구체화: 공장 노동자 하루, 한국어 메시지로 겪는 곤란, 임금 관련 불안
- [ ] **현장 관찰 미션**(2일차 투어): 키오스크·안내문·공공장소에서 "소외된 누군가가 겪는 불편" 관찰 → 사진·메모 (동의 하에)
- [ ] 가능하면 실제 대상(어르신/외국인/저문해)에게 **간단 인터뷰 1~2건**: "이런 글 받으면 어떻게 하세요?"
- [ ] 데모용 **시나리오 대사** 초안: 민수가 고지서 받는 장면, 마리아가 카톡 받는 장면

**산출물:** 페르소나 2장(1인 1페이지) + 관찰 메모/사진 + 시나리오 대사.
**저에게 넘길 것:** 앱에 넣을 **현실적인 예시 문구**(민수용 고지서 문장, 마리아용 사장님 카톡 문장).
**주의:** 관찰·인터뷰는 **반드시 동의** 받고, 개인정보(얼굴·이름) 노출 금지.

### C. UI·브랜드 디자인 🎨 (→ 완성도 15, 개발 직결)

**목표:** 앱 브랜드와 화면 룩앤필을 만들고, **제가 앱에 바로 넣을 수 있는 형태**로 전달.

**할 일**
- [ ] 로고 & 앱 이름 로고타입 "읽고" (Canva)
- [ ] 앱 아이콘 (1024px png)
- [ ] **컬러 팔레트** — 배경/글자/버튼 + 신호등 색(🟢🟡🔴). **고대비**(저시력·어르신 고려)
- [ ] 폰트 선정 — 큰 글씨·가독성 (예: Noto Sans KR / Pretendard)
- [ ] 핵심 화면 목업(발표·앱 참고용): ①홈 ②분석중 ③결과(요약+신호등) ④원문↔쉬운말 토글 ⑤할 일+캘린더 ⑥답장
- [ ] 접근성 가이드 1장: 최소 글자 크기, 터치 버튼 크기, 이모지 사용 규칙

**산출물:** 로고/아이콘 png, 목업 이미지, **디자인 토큰 문서**(색 HEX 코드, 폰트 이름, 글자 크기).
**저에게 넘길 것 (중요):**
- 색상 **HEX 코드**(배경/글자/버튼/신호등 3색)
- 폰트 이름(무료/오픈 라이선스로)
- 로고·아이콘 png 파일
- 화면 레이아웃 스케치(어디에 뭐가 오는지)
**주의:** 예쁨보다 **가독성 우선**(저문해·어르신 대상). 화면 하나에 정보 하나.

### D. 콘텐츠 & 60초 소개영상 🎬 (→ 발표 10)

**목표:** 발표 중 트는 **60초 영상**으로 감정을 건드리기.

**할 일**
- [ ] 스토리보드: "같은 벽, 같은 해방" — 민수(고지서)와 마리아(문자) 교차편집
- [ ] 나레이션 스크립트 (한국어 / 영어 자막)
- [ ] 영상 소스: 실사 촬영 or Canva AI 이미지 + 앱 화면 녹화(제가 데모 영상 제공)
- [ ] 편집(캡컷): 자막, 전환, BGM(저작권 free)
- [ ] 컷 흐름 예: 어려운 글에 막힘 → 읽고로 촬영 → 쉬운말/영어 + 신호등 → 얼굴이 풀림 → 로고 "읽고 — 어려운 글, 대신 읽고 알려드릴게요"

**산출물:** 60초 mp4 + 스크립트.
**저에게 넘길 것:** 영상에 넣을 **앱 화면 녹화가 필요한 장면 목록**(제가 찍어 드림).
**주의:** 60초 **엄수**. 자막은 크게.

### E. 발표 슬라이드 & 서비스 기획서 📑 (→ 발표 10 · AI 20)

**목표:** 7분 발표덱과 제출용 기획서. **AI 활용 내역**(필수 평가 항목) 정리.

**할 일 — 슬라이드 8장 내외**
- [ ] ① 문제 (통계 + 어려운 글 예시)
- [ ] ② 우리가 만난 사람 (민수 · 마리아)
- [ ] ③ 서비스 소개 (읽고 한 줄 + 화면)
- [ ] ④ 라이브 데모 안내 (여기서 실제 시연)
- [ ] ⑤ **AI 활용 흐름도** — "사진 1장 → AI 1번 호출 → 요약·풀이·위험·할 일·답장" (제가 내용 제공)
- [ ] ⑥ 사회적 가치 (두 사람에게 무엇이 달라지나)
- [ ] ⑦ 확장 비전 (어르신 사기예방 등, 같은 엔진)
- [ ] ⑧ 팀 소개 + AI 활용 도구(Claude·Canva·캡컷)
- [ ] 발표 대본(7분) + 시간 배분

**할 일 — 서비스 기획서** (제출물, `읽고_PRD.md` 기반 정리)
- [ ] 문제 정의 / 서비스 개요 / 기대효과 / **AI 활용 내역**(어떤 도구를 어떻게 썼는지)

**산출물:** 슬라이드(Canva) + 기획서(문서) + 발표 대본.
**저에게 넘길 것:** 슬라이드 ⑤ AI 흐름도 초안 검토 요청(제가 기술 내용 채워드림).
**주의:** 발표 구성은 **문제→사람→서비스→AI 활용→사회적 가치** 순서 권장.

### F. ★ 데모 준비물 (최우선, 누구든 지금 바로) 🗂️

**목표:** 제가 앱을 만들고 라이브 시연하려면 **실제 예시 자료**가 필요합니다. 이게 빨리 와야 개발이 빨라집니다.

**할 일**
- [ ] **실물 종이 고지서/안내문** 각 1장씩: 건강보험료, 과태료, 통신요금, (가능하면) 전세 재계약 안내문 — 개인정보는 가리고 촬영/스캔
- [ ] **한국어 문자·카톡 샘플** 3~4개(캡처): ①집합/업무 지시형("새벽 5시 집합, 늦으면 일당 깐다"류) ②임금/정산형 ③행정 안내(신고기한)형
- [ ] 각 샘플에 "정답"(사람이 이해한 핵심·기한·위험)을 메모로 첨부 → AI 결과 검증용
- [ ] 영어 데모 대비: 위 샘플이 영어로 잘 풀리는지 확인용 메모

**산출물:** 이미지 파일 모음(폴더) + 각 이미지 "정답" 메모.
**저에게 넘길 것:** 위 전부 (이걸로 AI 프롬프트를 맞춥니다).
**주의:** **개인정보(이름·주민번호·계좌·전화)는 반드시 가리고** 촬영. 공개된 샘플/본인 것 사용.

## 3. 제가(개발) 맡는 부분 — 참고용

- Flutter 앱 개발: 사진/캡처 입력 → AI 1회 호출 → 결과 화면(요약·토글·카드·신호등·할 일·캘린더·답장·음성).
- AI 연동(Claude API), 데모 안정화(폴백·비상 녹화용 화면 제공).
- 여러분께 필요한 것: **디자인 토큰(C), 예시 문구(A·B), 데모 실물 자료(F), 영상용 화면 녹화(D)**.
