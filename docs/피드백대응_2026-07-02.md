## 0. 한 줄 결론

**받은 피드백 4건 중 3건은 제품을 안 바꿔도 고쳐진다. 우리 PRD는 이미 좁고 뾰족한데, 발표에서 "전자문서"로 넓히면서 초점이 흐려졌다. 원래 PRD의 구체 프레이밍(고지서 + 문자·카톡)으로 돌아가고, 지속성 의심에는 형용사 대신 실제 비용 숫자로 답한다.**

피드백은 2개 뿌리에서 나온다.

| 뿌리 | 해당 피드백 | 진짜 문제 |
|---|---|---|
| 초점이 흐리다 | ② 전자문서 범위 · ③ 가치 한 문장 | 무엇을 푸는 앱인지 한 번에 안 꽂힘 |
| 지속 가능성을 못 믿겠다 | ① 지속성 · ④ 토큰 비용 | "데모는 되는데 계속 굴러가냐"는 의심 |

## 1. "지속성"을 어떻게 보여줄 건가 — 3층으로, 강한 순서로

각 층마다 **이미 코드/설계에 있는 증거**를 붙였다. 없는 걸 지어내지 않는다.

### (1) 제품 지속성 (Product)

- **누가·언제·얼마나:** 민수 = 고지서 올 때마다(월 수 회) / 마리아 = 사장님 문자·정산 있을 때마다(주 단위). **한국 생활문서는 계속 도착한다 → 반복 사용은 구조적으로 보장.**
- **한 번 쓰고 끝 아님:** 보관함 + **D-day 기한 뱃지 + 인앱 기한 알림**이 "돌아오는 이유". 기한 관리가 재방문 훅. (구현 완료 — `기능리뷰_CEO` A5·C2)
- **로드맵 1~2단계:** ① 모국어 확장(크메르/베트남/네팔 — 엔진은 이미 다국어 지원) → ② **어르신 사기예방(같은 엔진 재사용)**, 복지관·행정복지센터 비치.

### (2) 운영 지속성 (Operation)

- **운영 주체:** 백엔드가 **이미 Railway에 배포·라이브**(Spring Boot + Supabase). 데모용 목업이 아니라 굴러가는 인프라. 프롬프트·모델을 서버에서 중앙 관리.
- **콘텐츠 업데이트:** 위험 판정 로직·도움기관 연락처(1350/118/1345/1577)는 프롬프트 + 로컬 매핑 → **분기 단위 수동 업데이트로 충분**. 문서 종류가 늘어도 `ReadResult` 스키마가 고정이라 코드 변경 거의 없음.
- **피드백 루프:** ERD에 **`FEEDBACKS(is_helpful, reason)` 테이블이 이미 존재** → 오탐/정확도 사용자 피드백 수집 구조가 설계에 내장. "AI가 틀리면 사용자가 알려주고, 우리가 프롬프트를 고친다"가 말로 성립.

### (3) 비용 지속성 (Economics)

→ §4에서 실제 숫자로.

### 권고

**발표에서 지속성을 물으면 (3) 비용부터 답하라.** 심사가 대놓고 의심한 지점이라 숫자를 먼저 던지면 가장 크게 먹힌다. 그다음 (1) 반복 사용 → (2) 이미 운영 인프라 있음 순서.

## 2. "전자문서"가 너무 넓다 — 좁히지 말고, 원래 PRD로 돌아가라

**핵심:** 우리 PRD·기능정리본은 이미 좁다("고지서 / 문자·카톡"). "전자문서"는 발표에서 넓힌 말이다. 그 단어만 버리면 된다.

### 발표 금지어

"전자문서", "여러 가지 문서" → 쓰지 않는다. 항상 **구체 예시 2개**로만 말한다:

| | 문서 | 예시 | 페르소나/트랙 |
|---|---|---|---|
| 문서 A | 종이 고지서 | 건강보험료 · 주차 과태료 | 민수 (배리어프리) |
| 문서 B | 휴대폰 한국어 문자/카톡 | 사장님 임금 문자 · 스미싱 | 마리아 (다문화) |

### 왜 1개로 안 줄이고 2개를 사수하나

이 2개가 **두 트랙(배리어프리 + 다문화) 동시 출전의 근거**다. 1개로 줄이면 두 페르소나 서사가 무너진다. → **좁히되 2개 고정, 나머지 추상어는 삭제.**

> 초점은 더하기가 아니라 빼기다. 없앨 건 기능이 아니라 모호한 단어 하나("전자문서").

## 3. 추구하는 가치 — 한 문장

우리 문서에 이미 좋은 후보가 있다. 심사가 준 템플릿에 맞춰 다듬음.

**1문장 (정식):**
> 우리는 **한국 생활문서(고지서·문자)를 혼자 읽기 어려운 사람**이 **뜻·위험·기한을 놓쳐 손해 보는 것**을 **사진 한 장 → 쉬운 말 풀이 + 위험 신호등 + 할 일 안내**로 해결해서, **누구에게 묻지 않고 스스로 대응하게** 만든다.

**피치 라인 (대비형, 메인 슬로건):**
> "번역기는 글자를 바꿔줍니다. **읽고는 손해 보지 않게 다음 행동까지** 알려줍니다."

**한 호흡 (포스터/영상):**
> "어려운 한국 생활문서, 사진 한 장이면 대신 읽고 위험까지 짚어드려요."

## 4. "실제 토큰 얼마 나오냐" — 진짜 숫자

### 먼저 구조

읽고는 **요약과 추출을 따로 하지 않는다**. **1회 멀티모달 호출**로 OCR + 쉬운말/의역 + 항목 추출 + 위험 추론 + (native)답장 작문을 전부 처리한다. 이게 AI활용 20점의 핵심이자 **비용 효율의 근거**다(호출 1번에 5가지 작업).

### 모델·단가 (Claude, 100만 토큰당)

| 모델 | 입력 | 출력 | 용도 |
|---|---|---|---|
| **Opus 4.8** (`claude-opus-4-8`) | $5 | $25 | 데모(정확도 우선) |
| **Sonnet 5** (`claude-sonnet-5`) | $3 *(인트로 $2, ~2026-08-31)* | $15 *(인트로 $10)* | 운영 기본 |
| **Haiku 4.5** (`claude-haiku-4-5`) | $1 | $5 | 대량/저가 |

> ⚠️ 백엔드 `ANTHROPIC_MODEL`이 `claude-sonnet-4-6`으로 적혀 있음 → 최신은 **`claude-sonnet-5`**(같은 단가, 인트로가 더 쌈). env 한 줄만 갱신 권장.

### 1회 호출 토큰 추정 (1280px 다운스케일 이미지 기준)

- 입력 ≈ **2,200 토큰** = 시스템 프롬프트 ~600 + 이미지 ~1,500 + 파라미터 ~100
- 출력 ≈ **800 ~ 1,200 토큰** = 원문 + 풀이 + 카드 + 위험 + 할 일 (답장 포함 시 상단)

### 시나리오별 1회 비용 (환율 ₩1,400/$ 기준, 반올림)

| 시나리오 | 입력 | 출력 | Opus 4.8 | Sonnet 5(인트로) | Haiku 4.5 |
|---|---|---|---|---|---|
| 고지서 쉬운말 풀이 | ~2,200 | ~800 | **~₩43** ($0.031) | ~₩17 ($0.012) | ~₩9 ($0.006) |
| 문자 영어 + 답장 생성 | ~2,300 | ~1,200 | **~₩58** ($0.042) | ~₩23 ($0.017) | ~₩12 ($0.008) |

### 규모별 비용

| 단위 | 가정 | Opus 4.8 | Sonnet 5 | Haiku 4.5 |
|---|---|---|---|---|
| 사용자 1명 / 월 | 월 20건 | ~₩1,000 | ~₩400 | ~₩200 |
| 데모 라이브 | 심사 중 500회 | ~₩25,000 | ~₩10,000 | ~₩5,000 |

→ 발표 멘트: **"라이브 시연 비용은 무시할 수준이고, 운영은 모델 한 줄 교체로 3~7배 더 싸집니다."**

### 비용이 커지는 트리거 & 방어책

| 트리거 | 방어책 | 상태 |
|---|---|---|
| 이미지 해상도 ↑ → 입력 토큰 ↑ | 업로드 전 1280px 다운스케일 | 구현됨 |
| 출력 길이 ↑ (원문 긴 문서) | `max_tokens 1200` 캡 | 구현됨 |
| 호출 수 = 사용자 × 문서 | 선형 증가 (예측 가능) | — |
| 답장 톤 재생성 | 추가 1회 호출 = 추가 비용 | 사용자 명시 액션 |

### 유지 재원 / 절감 레버

- **모델 스왑:** 데모 Opus → 운영 Sonnet/Haiku (`ANTHROPIC_MODEL` env 1줄) = 3~7배 절감.
- **프롬프트 캐싱:** 고정 시스템 프롬프트 캐싱 시 반복분 입력비 최대 ~90% 절감.
- **재원:** 복지관·행정복지센터 B2G/파트너십, 사회공헌 후원.
- **실측 전환:** `ANALYSES` 테이블에 `model`·`latency_ms` 컬럼이 이미 있음 → **토큰 수 로깅 컬럼 1개 추가**하면 데모 후 실측치 확보. 발표에 "추정 ₩50, 실측 로깅 중"이라 하면 신뢰 ↑.

> 위 숫자는 **추정치**(심사도 "대략치" 요구). 정확히 하려면 실제 시스템 프롬프트로 `count_tokens` 1회 호출.
