## 0. 한 줄 결론

**앱은 거의 다 만들어졌다. 지금 가장 큰 위험은 "없는 기능"이 아니라 ① 데모가 네트워크에 이중으로 묶여 있다는 것(로그인 + 실서버), ② 있는 척하는 가짜 기능 2개(기한 알림, 권한 표시)다.** 남은 시간은 새 기능보다 "데모가 절대 안 죽게 + 이미 만든 것 완성"에 먼저 쓰고, 그다음 점수 대비 효율 좋은 소형 기능(할 일 체크, 예시 데모, 공유)을 얹는 순서를 권한다.

## 3. 빼거나 고쳐야 할 것 (신뢰를 깎는 순서대로)

### C1. 로그인 필수 게이트 — 데모의 첫 관문이 이메일+비밀번호 ⚠️ 최우선
- **현상:** 스플래시에서 세션 없으면 무조건 로그인 화면. 건너뛰기 없음. (PRD는 "계정/로그인 안 함"이 명시적 비목표였음)
- **왜 문제인가:** ① 발표 성공지표에 "심사위원 즉석 입력"이 있는데, 심사위원 폰에서 시연하려면 회원가입부터 해야 함. ② 더 아픈 건 제품 철학 모순 — **글 읽기 어려운 사람을 돕는 앱의 첫 화면이 이메일·비밀번호 6자 입력**이다. 김민수 페르소나가 여기서 막힌다. 심사위원이 이걸 지적하면 사회가치 서사가 흔들림.
- **권고 (택1):**
  - (a) **"로그인 없이 시작" 버튼 추가** — 게스트는 디바이스 ID로 동작, 보관함은 로컬. 백엔드 팀 작업은 그대로 보존. *(권장)*
  - (b) 최소한 데모 기기에 사전 로그인 + 발표 시나리오에서 로그인 화면 안 거치게. (심사위원 즉석 체험은 포기)
- **노력:** (a) 소~중 / (b) 0

### C2. 설정 "기한 알림" 토글 — 동작하지 않는 스위치
- **현상:** 켜고 꺼도 저장만 됨. 로컬 알림 패키지 자체가 프로젝트에 없음.
- **권고 (택1):** (a) 데모 빌드에서 **숨김** *(소, 안전)* / (b) `flutter_local_notifications` 붙여서 캘린더 추가 시 기한 전날 알림 진짜 등록 *(중 — 하면 "기한 관리" 서사가 완성되므로 시간 남으면 가치 있음)*
- **원칙:** 가짜 기능은 미구현보다 나쁘다. 심사위원이 켜보고 아무 일 없으면 완성도 점수로 직행.

### C3. 설정 "권한 · 모두 허용됨" 하드코딩
- **현상:** 실제 권한 상태와 무관하게 항상 "모두 허용됨" 초록 글씨.
- **권고:** `permission_handler`가 이미 있으니 실제 상태 읽어서 표시 *(소)*. 시간 없으면 상태 텍스트만 제거하고 "권한 설정 열기" 행으로.

### C4. 미사용 코드 (낮은 우선순위 — 발표에 안 보임)
- `lib/widgets/reply_draft_tile.dart` (share_plus 유일 사용처, 어디서도 안 씀), 구 위젯 몇 개, UI 없는 `regenerateReplies`.
- 심사에 영향 없으니 **시간 남을 때만** 정리. 단 `toggleAction`은 삭제 말 것 — A2에서 살린다.

## 4. 추가할 것 (점수 ÷ 노력 순)

> 노력 표기: 사람 기준 / Claude Code 기준. 전부 백엔드 변경 불필요(이미 API 있음).

### A1. 데모 폴백 캐시 부활 — 발표 보험 🥇 (완성15 + 발표10 직결)
- **현상:** 실서버 모드에서 분석 실패 → "연결 실패" 화면이 끝. PRD FR-20("재시도 1회 + 사전 캐시 폴백, 라이브 실패가 크래시/빈 화면이 안 됨")이 목 모드에만 사실상 존재.
- **제안:** 분석 호출 실패(재시도 후) 시 번들된 목 JSON 결과로 전환 + 상단에 "예시 결과예요" 배지 한 줄. 추가로 **데모 당일용 `USE_MOCK=true` 빌드를 별도로 뽑아두기** (와이파이 전멸 시나리오 대비).
- **노력:** 사람 반나절 / CC 20분

### A2. 할 일 체크박스 — 이미 만든 기능 마무리 (FR-14 MVP)
- **현상:** 결과 화면 할 일이 번호 리스트일 뿐 체크가 안 됨. 컨트롤러(`toggleAction`)와 서버 PATCH는 이미 있음.
- **왜:** "이해 → 행동 → **완료**"가 이 제품 서사의 마침표. 김민수가 '냈어요'에 체크하는 순간이 발표에서 제일 울림 있는 장면. 붙이는 건 UI 한 줄기.
- **노력:** 사람 2~3시간 / CC 10분

### A3. 홈 "예시로 해보기" — 심사위원 원탭 체험
- **제안:** 홈에 세 번째 카드 "예시로 해보기" → 번들 샘플(건강보험 고지서 / 과태료 / 사장님 카톡) 중 택1 → 즉시 결과 화면. 네트워크 무관, A1의 폴백 자산 재활용.
- **왜:** 심사위원은 자기 고지서를 안 들고 다님. 지금 구조에선 "직접 체험"이 사실상 불가능. 이 버튼 하나가 창의25·발표10에서 체험 장벽을 없앰. 부스 시연에서도 무한 반복 가능.
- **노력:** 사람 반나절 / CC 20분

### A4. 결과 공유 — "의존"을 "연결"로 (사회가치 서사)
- **제안:** 결과 화면에 공유 버튼 → 요약 한 문장 + 핵심 카드 + 할 일 텍스트를 `share_plus`(이미 의존성에 있고 현재 미사용)로 공유.
- **왜:** "매번 가족에게 물어봐야 하는 의존"이 문제 정의였는데, 공유는 그걸 "필요할 때 내가 선택해서 보여주는 연결"로 뒤집음. 발표 멘트 한 줄이 그냥 나옴.
- **노력:** 사람 2시간 / CC 10분

### A5. 기한 D-day 뱃지 — 홈 최근기록·보관함
- **제안:** `AnalysisSummary.cardDeadline`이 이미 내려옴 → "D-3" 같은 뱃지, 임박(≤7일)이면 강조색.
- **왜:** "기한을 놓쳐 연체되는 사람들" 문제와 직결. 데이터 이미 있고 계산만 하면 됨.
- **노력:** 사람 1~2시간 / CC 10분

### A6. 답장 재생성 + 톤 선택 — 라이브 AI를 한 번 더 보여주기
- **제안:** 답장 탭에 "다시 만들기 (정중하게/짧게)" 버튼 → 이미 구현된 `regenerateReplies(tone)` 호출.
- **왜:** AI활용 20점에서 "1회 호출"을 넘어 **AI가 사용자의 요구로 다시 일하는 장면**을 시연 가능. 마리아 시나리오에서 심사위원 앞 라이브로 보여주기 좋음. 백엔드·레포지토리 전부 완성 상태, UI만 없음.
- **노력:** 사람 반나절 / CC 20분

### A7. (선택) 전화 걸기 — FR-16
- 카드/할 일에 전화번호 있으면 `tel:` 버튼. 도움받을 곳 화면의 다이얼 코드 재사용.
- **노력:** 소. 시간 남으면.

## 5. 하지 말 것 (지금 시점에 함정인 확장)

| 후보 | 왜 지금 아님 |
|---|---|
| UI 전체 다국어(i18n) | 설정에 8개 언어 라벨이 있지만 UI는 한국어. 풀 i18n은 대형 작업 + 번역 검수 불가. 데모는 "풀이·답장 해설이 영어"로 충분. 발표에서 "엔진은 이미 8개 언어를 받는다" 멘트로 처리. (마리아가 한국어 설정 화면을 봐야 하는 아이러니는 알고는 있을 것 — 질문 들어오면 "UI 현지화는 로드맵" 답변 준비) |
| 음성 입력, 어르신 모드, 가족 리포트 | PRD Post-MVP. 발표 비전 슬라이드용이지 코드용 아님 |
| 보관함 검색·필터·페이징 고도화 | 데모에서 기록 3~5개뿐. 아무도 검색 안 함 |

## 7. 권장 실행 순서

```
1일차 (보험부터)
  ① A1 폴백 캐시 + USE_MOCK 데모 빌드   ← 발표가 절대 안 죽게
  ② C1 로그인 우회(게스트 or 사전로그인 결정)
  ③ C2·C3 가짜 토글 정리 (숨기기면 10분)

2일차 (점수 쌓기)
  ④ A2 할 일 체크박스
  ⑤ A3 예시로 해보기
  ⑥ A4 공유 + A5 D-day 뱃지

시간 남으면
  ⑦ A6 답장 재생성(톤) → AI활용 어필
  ⑧ C2(b) 진짜 로컬 알림 / A7 전화 걸기 / C4 데드코드 정리
```

**핵심 원칙 하나만 기억:** 심사위원 관점에서 이 앱의 승부처는 "기능 개수"가 아니라 **"라이브로 돌아가는데 절대 안 죽고, 눌러본 모든 것이 진짜로 동작한다"**는 신뢰다. 다른 팀이 Canva 목업일 때 우리 차별점이 '실제 구동'이라면, 실제 구동의 약점(네트워크 의존·가짜 토글)부터 지우는 게 차별점을 지키는 일이다.
